01

Scalars

A single number — the simplest building block

🌡️

Plain English: A scalar is just a regular number — your temperature (98.6°F), your age (25), a price ($4.99). It has size but no direction. Everything starts here.

What is a scalar?

A scalar is a single real number. Denoted by lowercase letters: a, b, λ, α. It represents magnitude alone — no direction.

a = 3.14  // a scalar
b = -2.7  // another scalar
c = a × b = -8.478  // scalar multiply

Scalars in AI — everywhere

Loss value 0.342 — how wrong is the model?
Learning rate α 0.001 — step size in training
Temperature 0.8 — randomness in LLM output
Dropout 0.2 — fraction of neurons off
Attention score 3.7 — how much to focus here

AI Application — Loss & Learning Rate

Training a neural network means minimizing a single scalar called the loss. MSE loss: L = (1/n)Σ(ŷ−y)². This one number drives the entire learning process. The learning rate α (also a scalar) controls how big each step is: w ← w − α × gradient. Too big → training explodes. Too small → training takes forever.

Vectors

Ordered lists of numbers — the language of data

🗺️

Plain English: A vector is like GPS directions — "3 blocks east, 2 blocks north." It has both a size AND a direction. In AI, every piece of data — an image, a word, a user profile — becomes a vector of numbers.

What is a vector?

An ordered list of n numbers. In 2D: [x, y]. In 768D: 768 numbers stacked together. Each number is a component.

v = [3, -1, 2]  // 3D vector
|v| = √(3²+1²+2²) = √14 ≈ 3.74  // magnitude

Vector Operations

Addition, scaling, normalization, and linear combinations

⚖️

Plain English: Think of vector addition as combining two walking directions. Scalar multiplication is like taking the same path but faster (×2) or slower (×0.5). Normalization is trimming the path down so it's exactly 1 unit long — same direction, standard length.

Addition

a + b = [a₁+b₁, a₂+b₂]
// Tip-to-tail geometrically
// Used in: weight updates
w ← w + (−α∇L)

Scalar multiply

c·v = [c·v₁, c·v₂]
// Stretch (c>1) or shrink (c<1)
// Negative c flips direction
// Used in: learning rate step

Normalize

v̂ = v / |v|
// Makes length = 1
// Keeps direction intact
// Used in: cosine similarity
Gradient descent = vector subtraction: Every training step does w_new = w_old − α∇L. This is adding a scaled vector (the gradient step) to the current weight vector. All of training is this — repeated millions of times on vectors with millions of dimensions.

AI Application — Linear Combinations Build Embeddings

Word2Vec's famous trick: king − man + woman ≈ queen. This is pure vector arithmetic — subtract the "man" direction vector, add the "woman" direction vector, and you land near "queen" in embedding space. Every LLM uses these linear combinations internally to blend concepts.

Residual connections in transformers: output = layer(x) + x — add the input vector directly to the transformed output. This vector addition is what makes deep networks (100+ layers) trainable.

Dot Product & Cosine Similarity

Measuring how much two vectors agree

🔦

Plain English: The dot product is like shining a flashlight — it measures how much vector A "casts a shadow" onto vector B. If they point the same way, the shadow is big (high dot product). If they're perpendicular, no shadow at all (zero). If opposite, negative shadow.

Formula

a·b = a₁b₁ + a₂b₂ + ... + aₙbₙ
// Also equals:
a·b = |a| × |b| × cos(θ)
// Cosine similarity (normalized):
sim(a,b) = (a·b) / (|a|×|b|)  // -1 to +1

AI Application — Attention & Semantic Search

The transformer's attention mechanism is entirely built on dot products: Attention = softmax(QKᵀ/√d). Every query vector is dotted against every key vector — the result is a similarity score telling the model which words to focus on.

Semantic search (like in RAG systems): convert your query to a vector, dot-product it against millions of document vectors, return top matches. This is why "How do I cook pasta?" finds "pasta cooking instructions" even without exact word matches — similar meaning = similar vectors = high dot product.

Norms & Distance

Measuring the "size" of a vector and distance between points

📏

Plain English: The norm is just the "length" of a vector — like measuring the diagonal of a rectangle. There are different ways to measure (like taking a straight line vs. walking along streets), and each gives a different number. AI uses L2 most, but L1 for sparsity.

Types of norms

L1 norm: ||v||₁ = |v₁|+|v₂|+...+|vₙ|  // Manhattan
L2 norm: ||v||₂ = √(v₁²+...+vₙ²)  // Euclidean ★
L∞ norm: ||v||∞ = max(|vᵢ|)  // max component

Distance metrics

MetricUsed in AI for
Euclidean ||a-b||₂k-NN, clustering, image similarity
Cosine 1-(a·b/|a||b|)Text similarity, embeddings
Manhattan ||a-b||₁Sparse features, LASSO

AI Application — Regularization

L2 regularization (Ridge/Weight Decay): Adds λ||w||₂² to the loss. Prevents weights from growing huge → simpler model → less overfitting. Used in virtually every modern neural network as "weight decay" in the optimizer.

L1 regularization (Lasso): Adds λ||w||₁ to the loss. Pushes many weights to exactly zero → sparse model. Used for feature selection. L1 produces sparse solutions because the L1 "ball" has corners — the minimum tends to land on a coordinate axis.

Linear Independence & Span

Which vectors carry new information vs. redundancy

🗂️

Plain English: Vectors are linearly independent if none of them can be made by combining the others. Think of ingredients — salt, pepper, and "half salt + half pepper" are NOT independent (the third is redundant). Independent vectors = unique directions = unique information.

Definitions

Independent: no vector = combination of others
Dependent: at least one is redundant

Span: all points reachable by combining vectors
Basis: minimal set spanning the full space
Standard 2D basis: e₁=[1,0], e₂=[0,1]
Any 2D point = a·e₁ + b·e₂

Why it matters for AI

A full-rank weight matrix has independent rows — it can transform data into any direction without losing information. A rank-deficient matrix collapses some dimensions → information loss.

PCA finds truly independent directions in your data. Feature engineering tries to remove redundant (dependent) features before training.

AI Application — Rank, Bottlenecks & LoRA

When a matrix has rank r, it only uses r independent directions — the rest are redundant. This is the mathematical basis for LoRA: researchers discovered that fine-tuning updates to large models have very low rank in practice. So instead of updating a full 4096×4096 matrix, they approximate it as A×B where A is 4096×4 and B is 4×4096 — using only 4 independent directions, not 4096.

Projection

Finding the shadow of one vector onto another

☀️

Plain English: Imagine shining a light straight down onto a vector. The shadow it casts on the ground is its projection. We measure "how much of vector A points in the direction of vector B?" That shadow length is the projection.

Formula

proj_b(a) = (a·b / |b|²) × b
// The vector component of a in direction b

scalar projection = a·b / |b|
// Just the length (signed number)

Geometric intuition

The projection of a onto b is the point on b's line that is closest to a. It's how much a "overlaps" with b. This is exactly what a neuron computes — how much does input x align with weight vector w?

AI Application — How a Neuron Fires

A single neuron computes output = w·x + b — this is a dot product, which is essentially projecting the input x onto the weight vector w. If the input aligns strongly with w (big projection), the neuron fires strongly. If perpendicular (no alignment), the neuron outputs ~0.

This is how neurons become feature detectors: a neuron whose weights form a pattern (like a horizontal edge) fires maximally when its input contains that exact pattern. The dot product measures how well the input matches the learned pattern.

Matrices

2D grids of numbers — the machinery of neural networks

🏭

Plain English: A matrix is a machine. You feed it a vector, it spits out a transformed vector. A 3×2 matrix takes 2 numbers in and outputs 3 numbers. Every neural network layer IS a matrix doing this transformation millions of times per second.

Structure & notation

A ∈ ℝ^(m×n)  // m rows, n columns
A[i][j] = element at row i, col j

Transpose: A^T flips rows↔cols
(A^T)[i][j] = A[j][i]
Shape: (m×n) → (n×m) after transpose

Special matrices in AI

TypeAI use
Identity ISkip/residual connections
Weight WEvery neural layer
Q, K, VTransformer attention
Embedding EWord → vector lookup
Covariance CPCA, batch norm

AI Application — GPT's Weight Matrices

GPT-3 has 175 billion parameters — almost all of them are numbers inside weight matrices. A single attention layer has four matrices (Q, K, V, output projection), each roughly 12,288 × 12,288 = 150M parameters. The forward pass is: input → multiply by Q matrix → multiply by K matrix → dot products → softmax → multiply by V matrix → multiply by output projection matrix. Four matrix operations per attention layer, 96 layers = 384 matrix multiplications per forward pass.

Matrix Multiplication

The single most important operation in deep learning

🔗

Plain English: To multiply two matrices, take each row of the first, dot it with each column of the second. The result is a new matrix of all those dot products. Think of it as computing "how much does each input dimension contribute to each output dimension?"

The rule

C = A × B
// A is (m×k), B is (k×n) → C is (m×n)
C[i][j] = Σₖ A[i][k] × B[k][j]

// Inner dimensions MUST match:
(m×k) × (k×n) → (m×n)
// NOT commutative: AB ≠ BA

Neural forward pass

// One layer:
z = W·x + b  // matrix × vector
a = σ(z)     // activation

// Full network (chain of matmuls):
h₁ = relu(W₁·x + b₁)
h₂ = relu(W₂·h₁ + b₂)
ŷ = softmax(W₃·h₂ + b₃)

Determinant & Inverse

How much a transformation scales area — and can it be undone?

📐

Plain English: The determinant tells you how much a matrix "squishes" or "stretches" space. If det=2, all areas double. If det=0, the matrix crushes everything flat into a lower dimension — like squashing a 3D cube into a 2D sheet — and this can NEVER be undone.

Determinant

det([[a,b],[c,d]]) = ad − bc

det > 0: preserves orientation
det < 0: flips orientation
|det| = area scale factor
det = 0: COLLAPSES to lower dim!
// Matrix is then NOT invertible

Inverse

A⁻¹: undoes transformation A
A × A⁻¹ = I (identity)

2×2 inverse:
A⁻¹ = (1/det) × [[d,-b],[-c,a]]

Solving Ax = b: x = A⁻¹b
// Only if det(A) ≠ 0
LoRA exploits low rank: The determinant of a rank-deficient matrix is 0. When researchers found that weight updates in fine-tuning LLMs are nearly rank-deficient (they don't need the full n² parameters), they created LoRA — approximate the update as A×B with far fewer parameters. This saves 99% of the compute for fine-tuning GPT-4-scale models.

Solving Ax = b

Systems of linear equations — the backbone of ML optimization

🧩

Plain English: "If I buy 2 apples and 3 bananas for $5, and 1 apple and 2 bananas for $3 — what does each fruit cost?" That's a system of linear equations. AI solves these (in millions of dimensions) to find optimal weights.

Methods

// Direct (small systems)
x = A⁻¹b  // if A is invertible

// Gaussian elimination
[A | b] → row reduce → solution

// Least squares (overdetermined)
x = (AᵀA)⁻¹Aᵀb  // normal equations
or solve: min ||Ax − b||₂²

Three possible outcomes

Unique solution — A is invertible (det≠0), exactly one x satisfies Ax=b. The system is perfectly determined.

No solution — overdetermined (more equations than unknowns). Use least squares — find x that gets as close as possible.

Infinite solutions — underdetermined (more unknowns than equations). Neural nets are always in this regime! Many weight combinations give zero training loss.

AI Application — Linear Regression & Normal Equations

Linear regression finds weights by solving: w = (XᵀX)⁻¹Xᵀy. This is the normal equations — the closed-form solution to the least squares problem min||Xw − y||². For small datasets, you can solve this exactly. For large datasets (n > 10,000), gradient descent is faster than computing the matrix inverse (which is O(n³)).

Rank & Null Space

How much information a matrix actually contains

🗜️

Plain English: The rank of a matrix is how many "truly different" dimensions of information it contains. A 1000×1000 matrix might have rank 50 — meaning it only really stores 50 independent pieces of information, not 1000. The null space is all the input vectors the matrix destroys (sends to zero).

Definitions

Rank(A) = #independent rows = #independent cols

Column space: all outputs A can produce
Null space: all x where Ax = 0

Rank + dim(Null space) = n columns
// Rank-nullity theorem

Eigenvectors & Eigenvalues

Special directions that only stretch, never rotate

🧭

Plain English: Most vectors get rotated AND scaled when you apply a matrix. Eigenvectors are special — they only get scaled (stretched or shrunk), never rotated. They're like the matrix's "natural axes." The eigenvalue is how much the scaling is. These special directions reveal the matrix's true structure.

Definition

Av = λv
// v = eigenvector (just scales, no rotation)
// λ = eigenvalue (scale factor)

Find eigenvalues: det(A − λI) = 0
Then for each λ: find v in (A−λI)v = 0

What eigenvalues tell you

λMeaning
Large λHigh variance direction in data
λ = 0Redundant direction (null space)
λ > 0 allPositive definite (bowl-shaped loss)
Some λ < 0Saddle point in loss surface!

AI Application — PageRank & Training Stability

Google PageRank: The importance of each webpage is the principal eigenvector (largest eigenvalue) of the web's link matrix. The most "important" page is the one that survives infinite iterations of following links — exactly what the dominant eigenvector captures.

Training stability: The largest eigenvalue of the loss Hessian determines the maximum safe learning rate. If α > 2/λ_max, gradients explode. This is why learning rate warmup and gradient clipping matter — they prevent the learning rate from exceeding this bound.

SVD — Singular Value Decomposition

The Swiss Army knife of matrix factorization

🔪

Plain English: SVD says every matrix = (rotation) × (stretch) × (rotation). Like decomposing a complex movement into: first rotate, then scale each axis independently, then rotate again. The stretching amounts are the singular values — sorted from biggest to smallest. Drop the small ones → compressed approximation.

Formula

A = U Σ Vᵀ

U (m×m): left singular vectors
Σ (m×n): diagonal, singular values σ₁≥σ₂≥...
V (n×n): right singular vectors

// Rank-k approximation:
Aₖ ≈ U[:,:k] Σ[:k,:k] Vᵀ[:k,:]
// Captures most of the info

SVD vs PCA

FeaturePCASVD
Works onSquare covarianceAny matrix
OutputEigenvectorsU, Σ, Vᵀ
StabilityCan be unstableNumerically stable
AI useVisualizationCompression, LoRA

AI Application — LoRA & Collaborative Filtering

LoRA (Low-Rank Adaptation): Fine-tuning GPT requires updating billions of parameters. LoRA instead approximates ΔW ≈ A×B (rank-r factorization via SVD insight) where r ≪ d. This reduces fine-tuning cost by 10,000×. Llama-3 can be fine-tuned on a single GPU with LoRA.

Netflix / Spotify recommendations: The user-item rating matrix is factored as U×Σ×Vᵀ. U encodes user tastes, V encodes item characteristics. Missing ratings predicted as Uᵢ·Vⱼ. This is collaborative filtering — pure SVD.

Making vectors perfectly perpendicular to each other

📌

Plain English: Imagine you have two arrows that aren't quite perpendicular. Gram-Schmidt makes them perfectly perpendicular by subtracting the "overlap" of one from the other. Like adjusting two rulers on a drafting table until they're exactly 90° apart. Orthogonal vectors don't interfere with each other — they're completely independent.

Gradient Vectors

The direction of steepest ascent — compass for optimization

🧭

Plain English: The gradient is like a compass needle that always points UPHILL on the loss surface. It's a vector of partial derivatives — each component says "if you nudge this weight by a tiny bit, the loss increases by this much." To minimize loss, we step in the OPPOSITE direction of the gradient.

What is a gradient?

∇L(w) = [∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ]
// A vector of all partial derivatives
// Points in direction of steepest INCREASE

To minimize: w ← w − α∇L
// Step OPPOSITE to gradient

Gradient field

At every point in weight space, the gradient points uphill. The loss surface is like a hilly landscape — the gradient at any location points toward the nearest peak (uphill). Training = following the gradient downhill to find the valley (minimum loss).

AI Application — Backprop Computes the Gradient Vector

Backpropagation efficiently computes ∇L with respect to ALL weights simultaneously using the chain rule. For a network with 175B parameters, it computes a 175-billion-dimensional gradient vector in one backward pass. PyTorch's autograd does this automatically — every operation is recorded in a computation graph so the gradient can be traced back.

Gradient Descent

How AI models actually learn — rolling downhill on the loss surface

⛷️

Plain English: Imagine you're blindfolded on a hilly mountain, trying to find the lowest valley. You feel the slope with your feet and take a step downhill. Repeat. That's gradient descent — each step makes the loss a bit smaller. The learning rate controls how big each step is. Too big and you overshoot the valley; too small and you take forever.

AI Application — SGD, Adam, and Why Learning Rate Matters

SGD uses one mini-batch's gradient per step — noisy but fast. Adam adapts the learning rate per parameter (scales the gradient by running average of squared gradients). This is why Adam converges faster — it effectively uses a different learning rate for each dimension of the parameter space.

Learning rate schedules (warmup + decay) start slow (avoids diverging early), ramp up (fast convergence), then decay (fine-tune near the minimum). State-of-the-art models use cosine annealing or linear warmup.

Backpropagation

Chain rule applied to matrix operations — how gradients flow backward

🔄

Plain English: After a prediction, backprop asks: "Whose fault was this error?" It traces blame backwards through the network — "the output was wrong, so the last layer's weights are 30% to blame, the second layer 50% to blame, etc." The chain rule does the math to figure out exactly how much each weight contributed to the error.

① Forward pass
② Compute loss
③ Backward pass
④ Update weights
Chain rule: ∂L/∂W₁ = ∂L/∂ŷ × ∂ŷ/∂h × ∂h/∂W₁
// Multiply gradients backwards through each layer

// Matrix form (for a linear layer):
∂L/∂W = δᵀ × x  // outer product
∂L/∂x = Wᵀ × δ  // gradient flows backwards
δ_prev = Wᵀ × δ × σ'(z)  // with activation

AI Application — Why PyTorch's Autograd is Magic

You never manually compute the chain rule. PyTorch records every operation in a computation graph and automatically differentiates through it. Calling loss.backward() triggers reverse-mode autodifferentiation — it walks the computation graph in reverse, applying the chain rule at each node, accumulating gradients into each weight's .grad attribute. This is backpropagation implemented generically for any computation graph.

Attention Mechanism

The core of transformers — built entirely from linear algebra

👁️

Plain English: When you read "The animal didn't cross the street because it was too tired", what does "it" refer to? You focus your attention on "animal". The attention mechanism does this mathematically — for each word, it computes dot products with every other word to figure out which ones to "pay attention to" when building its representation.

Interactive: Click a word to see what it attends to
Select a word above to see its attention weights across all other words.
Q = X × W_Q  // Query matrix: "what am I looking for?"
K = X × W_K  // Key matrix: "what do I contain?"
V = X × W_V  // Value matrix: "what info do I provide?"

Scores = Q × Kᵀ / √d_k  // dot products (similarity)
Weights = softmax(Scores)  // normalize to sum=1
Output = Weights × V  // weighted sum of values

AI Application — Transformers, GPT, BERT

Every transformer layer runs multi-head attention: run attention h times in parallel with different W_Q, W_K, W_V matrices, then concatenate and project. GPT-4 has 128 attention heads per layer, 128 layers. Each head learns different aspects: one tracks syntax, another tracks long-range dependencies, another tracks entity coreference. All learned automatically from data — just linear algebra and gradients.

Softmax

Turning any vector of scores into probabilities

🎲

Plain English: Softmax takes any list of numbers (could be negative, could be huge) and squeezes them into probabilities that sum to 1. The biggest number gets the most probability. Temperature controls how "confident" it is — low temperature = very sure (spiky), high temperature = uncertain (flat).

Interactive: Adjust logits and temperature
1.0
softmax(x)ᵢ = exp(xᵢ/T) / Σ exp(xⱼ/T)

AI Application — LLM Token Prediction

Every time GPT predicts the next word, it computes a score (logit) for every word in its 50,000-word vocabulary. Softmax converts these into probabilities. Then a token is sampled. Temperature controls creativity: T=0.1 always picks the most likely word (boring, repetitive), T=1.5 picks more surprisingly (creative, sometimes wrong). ChatGPT uses T≈0.7 for a good balance. The attention scores also go through softmax to ensure weights sum to 1.

Convolution as Matrix Multiplication

How CNNs detect features using sliding dot products

🔍

Plain English: A convolution is like sliding a magnifying glass (the kernel) over an image. At each position, you multiply the magnified pixels by the kernel's weights and sum up. A kernel for "vertical edges" will produce a big number wherever it finds a vertical edge. The kernel IS a learned feature detector.

Convolution = matrix multiply in disguise: The sliding kernel operation can be rewritten as im2col (arrange image patches as rows of a matrix) followed by a single matrix multiplication with the kernel. This is exactly how PyTorch/CUDA implements it for GPU acceleration.

AI Application — CNNs for Vision

A CNN's first layer might have 64 kernels, each 3×3×3 (RGB). Each kernel learns a different low-level feature: edges, colors, textures. Deeper layers combine these into faces, wheels, text. Each kernel application IS a dot product between the kernel vector and the image patch vector. A ResNet-50 does roughly 4 billion such dot products per image.

Vector Embeddings

Encoding meaning as geometry — the foundation of modern NLP

🌌

Plain English: What if you could place every word in a giant space where similar words are physically close to each other? "Cat" and "kitten" near each other. "King" near "queen" near "prince". "Run" and "sprint" close. That's embedding space — meaning becomes geometry, and you can do math on concepts.

AI Application — RAG, Recommendation & Semantic Search

RAG (Retrieval-Augmented Generation): Convert your question to a 1536D vector, search a database of millions of document vectors for highest cosine similarity → feed relevant docs as context to GPT. The entire retrieval step is a single large matrix multiplication (query vector vs. embedding matrix).

Recommendations: Netflix embeds users and movies in the same space. Your user vector drifts toward movies you liked. The nearest movie vectors are recommended. Matrix factorization (SVD) learns these embeddings from interaction history.
