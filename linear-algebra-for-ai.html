Why Linear Algebra?

The mathematical backbone of every AI system

The language of data

When a neural network processes an image, it doesn't "see" pixels — it sees a vector of numbers. When it generates text, it converts words to vectors. Every operation — attention, multiplication, gradient — is linear algebra.

Understanding these operations lets you reason about what AI models are actually doing under the hood, debug them, and design better architectures.

Where it appears in AI

Neural networks Transformers PCA / Dimensionality reduction Embeddings Backpropagation Attention mechanism SVD / Recommendations Computer vision NLP / LLMs

Big picture: A neural network is just a sequence of matrix multiplications followed by nonlinear functions. The entire art of deep learning is about choosing the right matrices (weights), finding them via gradient descent, and composing them cleverly.
01

Scalars

Single numbers — the atomic unit of math

What is a scalar?

A scalar is simply a single real number. It has magnitude but no direction. Examples: temperature (98.6°F), price ($4.99), a model's loss (0.342).

Scalars are denoted by lowercase letters in italics: a, b, λ, α.

a = 3.14   // a scalar
b = -2.7   // another scalar
c = a × b = -8.478   // scalar multiplication

In AI: where scalars appear

Loss value — a single number measuring how wrong the model is (e.g., 0.342).
Learning rate α — how big each gradient step is (e.g., 0.001).
Temperature — controls randomness in LLM outputs.
Dropout rate — fraction of neurons to turn off (e.g., 0.2).
Attention score — a single similarity number between two tokens.

AI Application — Loss Function

Training a neural network means minimizing a scalar called the loss. For example, Mean Squared Error: L = (1/n) Σ (ŷᵢ - yᵢ)². This single number tells us how far off our predictions are. The entire backpropagation algorithm works backwards from this one scalar to update millions of weights.

The learning rate α (also a scalar) controls how aggressively we move: w ← w - α × ∂L/∂w. Too large and training diverges; too small and it never converges.

Loss: 0.342 α = 0.001 Temperature = 0.8 Epochs = 100

Ordered lists of numbers — representing data as points in space

What is a vector?

A vector is an ordered list of numbers arranged in a column. Each number is a component. A 2D vector lives on a plane; a 3D vector in space; an 768D vector in an embedding space.

AI Application — Data Representation

In machine learning, every data point is a vector. A tabular dataset row is a feature vector. An image is a pixel vector. A sentence in an LLM is a sequence of token vectors, each with 768–4096 dimensions.

The key insight: by representing data as vectors, we can use all the tools of linear algebra — distance, similarity, transformation — to process, compare, and classify that data at massive scale.

03

Vector Operations

Addition, subtraction, scaling, and projection

Core operations

// Addition (component-wise)
a + b = [a₁+b₁, a₂+b₂, ..., aₙ+bₙ]

// Scalar multiplication
c·v = [c·v₁, c·v₂, ..., c·vₙ]

// Normalization (unit vector)
v̂ = v / |v|   // length = 1

// Linear combination
αa + βb = new vector

Geometric intuition

Addition: Place the tail of B at the tip of A. The result points to the new tip. Used in gradient updates — the new weight vector = old vector + gradient step.

Scalar multiplication: Stretches (|c|>1) or shrinks (|c|<1) the vector. Negative c reverses direction.

Normalization: Makes a vector have length 1 while preserving direction. Critical for cosine similarity and attention.

Gradient descent is vector addition: At each training step, the new weights are computed as w_new = w_old + (-α × ∇L) — the old weight vector plus a scaled gradient vector. All of model training is repeated vector addition in millions of dimensions.

Linear Independence

Vectors are linearly independent if no vector can be written as a combination of the others. Independent vectors span a full space. Dependent vectors are "redundant."

In neural nets, you want learned features to be independent — capturing different aspects of data.

Span & Basis

The span of a set of vectors is all possible linear combinations. A basis is a minimal spanning set — enough vectors to reach every point in a space.

Standard basis in 2D: e₁=[1,0], e₂=[0,1]. Any 2D vector = a·e₁ + b·e₂.

Projection

The projection of a onto b is how much of a points in the direction of b:

proj_b(a) = (a·b / |b|²) × b

Used in attention and PCA to find components along specific directions.

AI Application — Gradient Descent

Every neural network training step is a vector operation. The gradient ∇L is a vector pointing uphill in loss space. We subtract a scaled version of it: w ← w − α∇L. This is vector subtraction, performed millions of times, on weight vectors with millions of components (parameters).

Adam optimizer adds momentum (another vector) and adaptive scaling (yet another vector) to make this descent faster and more stable. It's all vectors.

Dot Product & Cosine Similarity

Measuring alignment and similarity between vectors

What is a dot product?

Multiply corresponding components, then sum everything up. The result is a single scalar.

a · b = a₁b₁ + a₂b₂ + ... + aₙbₙ

// Geometric form
a · b = |a| × |b| × cos(θ)

// Cosine similarity (normalized)
cos(θ) = (a·b) / (|a| × |b|)

AI Application — Attention Mechanism (Transformers)

The attention mechanism in GPT/BERT is built entirely on dot products. For each token, we compute Attention(Q,K,V) = softmax(QKᵀ / √d_k) × V. The QKᵀ step is a giant matrix of dot products — each query vector dotted against every key vector — producing a similarity score that determines how much attention to pay.

Vector search (used in RAG, recommendation systems) works the same way: find the top-K vectors with the highest cosine similarity to your query. FAISS, Pinecone, and Weaviate all implement fast dot-product search over millions of vectors.

Norms & Distance

Measuring vector length and distance between points

Vector norms

A norm measures the "size" or "length" of a vector. Different norms measure size differently.

// L1 norm (Manhattan distance)
||v||₁ = |v₁| + |v₂| + ... + |vₙ|

// L2 norm (Euclidean length) ← most common
||v||₂ = √(v₁² + v₂² + ... + vₙ²)

// L∞ norm (max component)
||v||∞ = max(|v₁|, |v₂|, ..., |vₙ|)

// General Lp norm
||v||ₚ = (|v₁|ᵖ + |v₂|ᵖ + ... + |vₙ|ᵖ)^(1/p)

Distance metrics

Euclidean||a-b||₂k-NN, clustering
Cosine1 - (a·b)/(|a||b|)Text similarity
Manhattan||a-b||₁Sparse features

AI Application — Regularization & Loss Functions

Norms appear everywhere in AI training. L2 regularization (Ridge) adds λ||w||₂² to the loss to prevent weights from growing too large — keeping the model simple. L1 regularization (Lasso) adds λ||w||₁ and produces sparse weights (many exactly zero) — useful for feature selection.

Distance metrics determine how similar data points are in k-NN classification, DBSCAN clustering, and embedding search. Choosing the right metric for your data type can make or break a model's performance.

Matrices

2D arrays of numbers — the building blocks of neural networks

What is a matrix?

A matrix is a 2D grid of numbers with m rows and n columns. We write it as A ∈ ℝ^(m×n). Each element Aᵢⱼ is at row i, column j.

A = [[a₁₁, a₁₂, a₁₃], // row 1
     [a₂₁, a₂₂, a₂₃], // row 2
     [a₃₁, a₃₂, a₃₃]] // row 3

Shape: 3×3   // m rows × n columns

Special matrix types

The transpose flips rows and columns. If A is m×n, then Aᵀ is n×m. In transformers, the operation QKᵀ transposes K before multiplying — this is how attention scores are computed across all token pairs simultaneously.

AI Application — Weight Matrices in Neural Networks

Every layer in a neural network is a weight matrix W ∈ ℝ^(output × input). A layer that takes 512 inputs and produces 1024 outputs has a weight matrix with shape (1024 × 512) = 524,288 parameters. GPT-4 has roughly 1.8 trillion such parameters across its layers.

The forward pass is: output = W × input + bias. The backward pass computes gradients with respect to W and updates it. Every second of ChatGPT inference involves thousands of matrix multiplications happening on specialized GPU hardware (CUDA cores).

Matrix Multiplication

The fundamental operation of deep learning

How matrix multiplication works

To multiply A (m×k) by B (k×n): the result C is m×n. Each element Cᵢⱼ is the dot product of row i of A with column j of B.

Cᵢⱼ = Σₖ Aᵢₖ × Bₖⱼ

// Key rule: inner dimensions must match
(m×k) × (k×n) → (m×n)

// NOT commutative!
AB ≠ BA in general

Neural network forward pass

// One layer computation
z = W × x + b
a = σ(z)   // activation function

// Stacked layers (chain of matmuls)
h₁ = σ(W₁x + b₁)
h₂ = σ(W₂h₁ + b₂)
ŷ = softmax(W₃h₂ + b₃)

Batched matrix multiplication: In practice, we process many inputs at once. If we have a batch of 32 inputs each with 512 features, we pass a matrix X (32×512) through a weight matrix W (512×1024) to get output (32×1024). GPUs are specifically optimized for exactly this — running thousands of dot products in parallel.

AI Application — Why GPUs Rule AI

A GPU has thousands of cores designed to run the same operation on many data simultaneously (SIMD). Matrix multiplication is the perfect workload because every element Cᵢⱼ = row·column can be computed independently in parallel. Training GPT-3 required 3.14 × 10²³ floating-point operations — almost all matrix multiplications.

Hardware companies (NVIDIA, Google, Cerebras) are building chips specifically to accelerate these ops. Google's TPUs can do 420 trillion matrix ops per second per chip.

Linear Transformations

How matrices reshape, rotate, and compress data

What makes a transformation linear?

A transformation T is linear if it satisfies two properties:

// Additivity
T(u + v) = T(u) + T(v)

// Homogeneity
T(c·v) = c·T(v)

// Every linear transformation is a matrix!
T(x) = Ax   for some matrix A

AI Application — What Neural Network Layers Learn

Each layer of a neural network applies a linear transformation (matrix multiply) followed by a nonlinear activation function. The transformation rotates, scales, and reshapes the data in high-dimensional space to make classes linearly separable. Deeper networks can carve more complex boundaries through successive transformations.

This is why deep networks are powerful: with enough transformations, any curved decision boundary can be approximated. The weight matrix in each layer is literally a transformation matrix being optimized to separate data correctly.

Determinant & Matrix Inverse

Measuring transformation strength and reversibility

Determinant

The determinant of a matrix is a scalar that encodes how a transformation scales area (2D) or volume (3D).

// 2×2 determinant
det([[a,b],[c,d]]) = ad − bc

// What it means
det > 0 : orientation preserved
det < 0 : orientation flipped
det = 0 : collapses to lower dimension!
|det| > 1 : area expands
|det| < 1 : area shrinks

Matrix Inverse

The inverse A⁻¹ undoes the transformation A. A × A⁻¹ = I (identity).

// 2×2 inverse formula
A⁻¹ = (1/det(A)) × [[d,-b],[-c,a]]

// Only exists when det(A) ≠ 0
Solving Ax = b:
x = A⁻¹b   // if A is invertible

// Pseudo-inverse for non-square
A⁺ = (AᵀA)⁻¹ Aᵀ   // Moore-Penrose
Rank matters: The rank of a matrix is the number of linearly independent rows (or columns). A full-rank matrix can represent information without loss. Low-rank matrices compress information — used in LoRA (Low-Rank Adaptation), a method for efficiently fine-tuning LLMs with far fewer parameters.

AI Application — LoRA: Low-Rank Adaptation of LLMs

Fine-tuning GPT-3 requires updating 175 billion parameters — too expensive. LoRA instead represents weight updates as low-rank matrices: ΔW = A × B where A is (d×r) and B is (r×d), with r ≪ d. Instead of updating a 4096×4096 matrix (16M params), we update two small matrices totaling just thousands of params.

This works because weight updates tend to be low-rank in practice. The determinant/rank insight is what makes this possible.

Eigenvectors & Eigenvalues

Special vectors that only stretch — the key to PCA and stability

What are eigenvectors?

An eigenvector of a matrix A is a special vector that, when transformed by A, doesn't change direction — it only gets scaled. The scale factor is the eigenvalue λ.

Av = λv

// v is an eigenvector, λ is its eigenvalue
// Most vectors get rotated AND scaled
// Eigenvectors ONLY get scaled (not rotated)

(A − λI)v = 0
det(A − λI) = 0  // characteristic equation

What eigenvalues tell you

λ valueMeaningAI context
λ > 1Direction expandsHigh-variance feature
0 < λ < 1Direction shrinksLow-variance feature
λ = 0Direction collapsesRedundant feature
λ < 0Direction flipsAnticorrelated feature
Large λMost important directionFirst principal component

PCA: Principal Component Analysis

PCA finds the eigenvectors of the data's covariance matrix. These eigenvectors point in the directions of maximum variance. By projecting data onto the top K eigenvectors (principal components), we compress data with minimal information loss.

// PCA algorithm
1. Center data: X̄ = X − mean(X)
2. Covariance: C = (1/n) X̄ᵀX̄
3. Eigendecompose: C = VΛVᵀ
4. Sort by eigenvalue (largest first)
5. Project: Z = X̄ × V[:, :k]   // keep top k components

Each principal component is an eigenvector. The eigenvalue tells you how much variance that component explains. PCA is used to visualize high-dimensional data (project 768D embeddings to 2D for plotting) and remove noise.

AI Application — PCA, t-SNE, and Stability

Dimensionality reduction (PCA, t-SNE, UMAP) uses eigenvectors to compress data. When you see beautiful 2D "clusters" of word embeddings — that's PCA projecting 768D vectors to 2D using the top 2 eigenvectors of the covariance matrix.

Gradient stability: The largest eigenvalue of the Hessian matrix determines the maximum safe learning rate. If your learning rate exceeds 2/λ_max, training explodes. Understanding eigenvalues helps diagnose training instability.

PageRank (Google): Finding the most important webpage is an eigenvector problem — the web link matrix's principal eigenvector gives page importance scores.

Representing meaning as points in high-dimensional space

What are embeddings?

An embedding is a dense vector representation learned from data. Instead of arbitrary IDs, similar items get similar vectors. The geometry of embedding space encodes semantic meaning.

AI Application — RAG, Semantic Search & Recommendation

RAG (Retrieval-Augmented Generation): When you ask ChatGPT a question, it may first embed your query as a vector, search a database of document vectors for the closest matches (using cosine similarity), and feed those documents as context. This is purely linear algebra — query vector vs database of vectors.

Recommendation systems: Netflix, Spotify, Amazon embed users and items as vectors. Items close to your user vector get recommended. The embedding space is learned from your interaction history. Adding vectors: user vector + "action movie genre" vector → new user vector → recommendations shift.

SVD & Matrix Decomposition

Factoring matrices to reveal hidden structure

Singular Value Decomposition (SVD)

SVD factorizes any matrix A into three matrices. It's the most important decomposition in numerical linear algebra and AI.

A = U Σ Vᵀ

// U (m×m): left singular vectors (input space)
// Σ (m×n): diagonal, singular values (importance)
// V (n×n): right singular vectors (output space)

// Truncated SVD (rank-k approximation)
Aₖ = U[:,:k] Σ[:k,:k] Vᵀ[:k,:]

Eigendecomposition vs SVD

SVD for Image Compression

A grayscale image is a matrix. SVD decomposes it into layers of decreasing importance. By keeping only the top k singular values, we get a compressed approximation. A 1000×1000 image (1M values) compressed with rank-50 SVD needs only 50×(1000+1000+1) ≈ 100K values — 90% smaller with visually acceptable quality. This same principle underlies LoRA, matrix factorization, and recommendation systems.

// Full rank: store 1000×1000 = 1,000,000 values
// Rank-50: store U(1000×50) + Σ(50) + V(1000×50)
Storage: 100,050 values   // 90% compression

QR Decomposition

A = QR where Q is orthogonal (Qᵀ=Q⁻¹) and R is upper triangular. Used to solve least squares problems efficiently — the foundation of linear regression.

LU Decomposition

A = LU where L is lower triangular and U is upper triangular. Efficiently solves systems Ax = b via forward/backward substitution. O(n³) once, then O(n²) per solve.

Cholesky

A = LLᵀ for symmetric positive-definite matrices. 2× faster than LU. Used in Gaussian processes, Kalman filters, and MCMC sampling in Bayesian ML.

AI Application — Collaborative Filtering & Matrix Factorization

Netflix's recommendation system uses matrix factorization (essentially truncated SVD). The user-item rating matrix R (users × movies) is factored as R ≈ U × Vᵀ where U is (users × k) and V is (movies × k). The k latent factors capture hidden concepts like "action lover" or "comedy preference." Missing ratings are predicted by U[user] · V[movie]ᵀ.

Transformers use SVD implicitly: The attention weight matrix has low effective rank — most information lies in a few singular vectors. LoRA exploits this by approximating weight updates as A×B (rank decomposition).

Matrix factorization (Netflix) LoRA (LLM fine-tuning) Linear regression (QR) Gaussian processes (Cholesky)
