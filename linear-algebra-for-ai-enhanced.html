Scalars

A single number — the simplest building block

🌡️

Plain English: A scalar is just a regular number — your temperature (98.6°F), your age (25), a price ($4.99). It has size but no direction. Everything starts here.

What is a scalar?

A scalar is a single real number. Denoted by lowercase letters: a, b, λ, α. It represents magnitude alone — no direction.

a = 3.14  // a scalar
b = -2.7  // another scalar
c = a × b = -8.478  // scalar multiply

Scalars in AI — everywhere

Loss value 0.342 — how wrong is the model?
Learning rate α 0.001 — step size in training
Temperature 0.8 — randomness in LLM output
Dropout 0.2 — fraction of neurons off
Attention score 3.7 — how much to focus here

AI Application — Loss & Learning Rate

Training a neural network means minimizing a single scalar called the loss. MSE loss: L = (1/n)Σ(ŷ−y)². This one number drives the entire learning process. The learning rate α (also a scalar) controls how big each step is: w ← w − α × gradient. Too big → training explodes. Too small → training takes forever.

Deep Dive — Scalars in Practice

concept · why · how · example
1The Concept

A scalar is a single real number with magnitude only — no direction, no list, no structure. Mathematically, scalars live in ℝ (the real number line). They are the simplest object in linear algebra and serve as the "raw material" everything else is built from: a vector is a list of scalars, a matrix is a grid of scalars, a tensor is a higher-dimensional cube of scalars.

Scalars do two jobs in AI: they measure things (loss, accuracy, probability) and they control things (learning rate, temperature, dropout). Anywhere you see a single number governing behavior, you're looking at a scalar.
2Why It Matters in AI

Optimization needs a single number to minimize. You can't optimize "make the model better" — you have to compress the entire idea of "better" into one scalar (the loss). Backpropagation computes the gradient of this scalar with respect to every weight, and gradient descent uses the scalar learning rate α to scale each update step. Even a 175-billion-parameter model is ultimately driven by one scalar at the top.

3How It's Used — Step by Step
  1. Pick a loss function that maps predictions to a scalar — e.g., MSE: L = (1/n) Σ (ŷᵢ − yᵢ)².
  2. Run a forward pass on a mini-batch. The model outputs predictions ŷ, and the loss collapses everything into one number.
  3. Compute the gradient of that scalar with respect to every weight: ∇L = ∂L/∂w.
  4. Update each weight using the scalar learning rate: w ← w − α × ∇L.
  5. Repeat for thousands of mini-batches. The scalar loss should trend downward over time.
4Worked Example

Suppose a model predicts house prices. For 3 houses the true prices are y = [200, 300, 250] (thousands $) and the model predicts ŷ = [210, 290, 260].

Giveny = [200, 300, 250], ŷ = [210, 290, 260]
Step 1errors = ŷ − y = [10, −10, 10]

Step 3sum = 300; mean = 300 / 3 = 100
ResultL = 100 # one scalar drives the entire training step
Updatew ← w − 0.01 × ∇L # α=0.01 is also a scalar

Deep Dive — Why Everything in AI Becomes a Vector

concept · why · how · example
1The Concept

A vector is an ordered list of n scalars, written as v = [v₁, v₂, …, vₙ]. It lives in n-dimensional space ℝⁿ. Two interpretations matter: (a) a vector is a point at coordinates (v₁, …, vₙ), or (b) a vector is an arrow from the origin to that point with magnitude |v| = √(v₁²+…+vₙ²) and direction. AI uses both interpretations interchangeably depending on context.

"Order matters." [3, 2] and [2, 3] are different vectors pointing in different directions. The position of each number encodes which feature it represents — slot 1 is always age, slot 2 is always income, slot 3 is always credit score, and so on.
2Why It Matters in AI

Computers work with numbers, not concepts. To process a sentence, photo, or user, AI must first encode that thing as a vector. Every input is converted into a fixed-length list of numbers — a 224×224 image becomes a 150,528-D pixel vector, a sentence becomes a sequence of 768-D embedding vectors, a user's profile becomes a feature vector. Once everything lives in vector form, the same mathematical machinery (dot products, distances, matrix multiplies) handles all data types uniformly.

3How It's Used — Step by Step
  1. Tokenize / featurize the raw input. For text: split into words. For images: lay out pixel intensities. For users: pick measurable attributes (age, clicks, spend).
  2. Map to numbers. Each token/feature becomes a scalar — either the raw value (pixel intensity 0–255) or a learned embedding (word "cat" → [0.21, −0.45, 0.13, …]).
  3. Stack into a fixed-length vector. Every example must end up in ℝⁿ for the same n, so the model's input layer has a known shape.
  4. Feed into the model. The first layer multiplies this vector by a weight matrix: z = Wx + b — every neural network starts with vector inputs.
4Worked Example

Encode a user as a vector for a churn-prediction model. Features: age, monthly spend ($), days since last login.

Addition, scaling, normalization, and linear combinations

⚖️

Plain English: Think of vector addition as combining two walking directions. Scalar multiplication is like taking the same path but faster (×2) or slower (×0.5). Normalization is trimming the path down so it's exactly 1 unit long — same direction, standard length.

Deep Dive — Combining Vectors to Build New Meaning

concept · why · how · example
1The Concept

Three operations cover almost everything you do to vectors: addition (combine two vectors element-wise), scalar multiplication (stretch or shrink while keeping direction), and normalization (rescale to unit length). Together these define linear combinations: c₁·v₁ + c₂·v₂ + … + cₙ·vₙ. Any new vector you can possibly produce by mixing existing ones in this way lives in their span.

Linear combinations are the only "moves" allowed in linear algebra. Add. Scale. Combine. Everything else — projections, transformations, even neural layers — is built from these three primitives applied at scale.
2Why It Matters in AI

Every weight update is a vector addition: w_new = w_old + Δw. Every neuron output is a scaled sum of inputs: z = w₁x₁ + w₂x₂ + … + wₙxₙ — that's a linear combination of the input components weighted by w. Residual connections in Transformers and ResNet are literal vector additions: output = layer(x) + x. Without that simple "+x", deep networks couldn't be trained because gradients would vanish through 100+ layers.

3How It's Used — Step by Step
  1. Compute the gradient ∇L — itself a vector pointing uphill on the loss surface.
  2. Scale it by the learning rate: Δw = −α · ∇L (negative because we want to go downhill).
  3. Add to current weights: w ← w + Δw. This is pure vector addition.
  4. For normalization (e.g. embeddings): compute |v|, then divide each component: v̂ = v / |v|. Now |v̂| = 1, direction preserved.

Deep Dive — The Dot Product is the Soul of Modern AI

concept · why · how · example
1The Concept

The dot product takes two vectors of the same length and returns one scalar: a·b = a₁b₁ + a₂b₂ + … + aₙbₙ. Geometrically, this equals |a|·|b|·cos(θ) where θ is the angle between them. So a dot product encodes two things at once — how big the vectors are, and how aligned they are. Divide by the magnitudes and you isolate alignment alone: that's cosine similarity, ranging from −1 (opposite) through 0 (perpendicular / unrelated) to +1 (identical direction).

If vectors encode meaning, then "similar meaning" = "small angle between vectors" = "high dot product." This single insight powers search engines, recommendation systems, and every Transformer ever built.
2Why It Matters in AI

The dot product is the universal similarity measure. A neuron's output is one giant dot product (input · weights) — it asks "how much does this input look like the pattern I've learned?" Transformer attention computes one dot product for every pair of tokens (Q · K) — it asks "which words should this word pay attention to?" Vector-database search returns the documents whose embedding has the highest dot product with your query. Modern GPUs are essentially specialized hardware for doing trillions of dot products per second.

3How It's Used — Step by Step
  1. Encode both items as vectors in the same space and dimension (query and document, two words, two users).
  2. Multiply element-wise and sum: a·b = Σ aᵢbᵢ.
  3. Normalize (optional but common): divide by |a|·|b| to get cosine similarity. This removes magnitude effects so length doesn't bias results.
  4. Interpret: the larger the score, the more aligned. For ranking, sort candidates by dot product descending and take the top-k.

Norms & Distance

Measuring the "size" of a vector and distance between points

📏

Plain English: The norm is just the "length" of a vector — like measuring the diagonal of a rectangle. There are different ways to measure (like taking a straight line vs. walking along streets), and each gives a different number. AI uses L2 most, but L1 for sparsity.

Types of norms

L1 norm: ||v||₁ = |v₁|+|v₂|+...+|vₙ|  // Manhattan
L2 norm: ||v||₂ = √(v₁²+...+vₙ²)  // Euclidean ★
L∞ norm: ||v||∞ = max(|vᵢ|)  // max component

Distance metrics

MetricUsed in AI for
Euclidean ||a-b||₂k-NN, clustering, image similarity
Cosine 1-(a·b/|a||b|)Text similarity, embeddings
Manhattan ||a-b||₁Sparse features, LASSO

AI Application — Regularization

L2 regularization (Ridge/Weight Decay): Adds λ||w||₂² to the loss. Prevents weights from growing huge → simpler model → less overfitting. Used in virtually every modern neural network as "weight decay" in the optimizer.

L1 regularization (Lasso): Adds λ||w||₁ to the loss. Pushes many weights to exactly zero → sparse model. Used for feature selection. L1 produces sparse solutions because the L1 "ball" has corners — the minimum tends to land on a coordinate axis.

Deep Dive — Different Ways to Measure "Length"

concept · why · how · example
1The Concept

A norm is any function that assigns a non-negative "size" to a vector and follows three rules: zero vector has zero norm, scaling a vector scales its norm proportionally, and the triangle inequality holds. Different norms emphasize different things. L2 (Euclidean) is the straight-line distance you learned in school. L1 (Manhattan) sums absolute values — like walking on a city grid where you can only go north/south/east/west. L∞ takes the largest absolute component and ignores the rest.

Norms also define distance: the distance between two points a and b is just ||a − b||. The choice of norm changes which points feel "close" — Manhattan thinks (3,0) and (0,3) are equally close to the origin, while Euclidean considers diagonal points closer.
2Why It Matters in AI

Norms appear in three places: (a) as regularizers added to the loss to prevent overfitting — L2 penalizes large weights smoothly, L1 pushes weights to exactly zero (creating sparse models); (b) as distance metrics in nearest-neighbor methods, clustering, and retrieval; (c) as safety bounds — gradient clipping rescales the gradient when its L2 norm exceeds a threshold, preventing exploding-gradient training failures.

3How It's Used — Step by Step
  1. For regularization: compute ||w||² = Σ wᵢ² across all model weights, multiply by hyperparameter λ, and add it to the loss: L_total = L_task + λ·||w||².
  2. For gradient clipping: compute g_norm = ||∇L||₂; if it exceeds a threshold T, rescale: ∇L ← (T / g_norm) · ∇L. This caps each step's magnitude.
  3. For nearest-neighbor search: for each candidate c, compute ||q − c||, sort ascending, return the closest k.
  4. For L1 sparsity: add λ · Σ |wᵢ| to the loss. The non-smooth shape of L1 at zero pushes many weights to exactly 0, yielding a sparse model.
4Worked Example

Linear Independence & Span

Which vectors carry new information vs. redundancy

🗂️

Plain English: Vectors are linearly independent if none of them can be made by combining the others. Think of ingredients — salt, pepper, and "half salt + half pepper" are NOT independent (the third is redundant). Independent vectors = unique directions = unique information.

Definitions

Independent: no vector = combination of others
Dependent: at least one is redundant

Span: all points reachable by combining vectors
Basis: minimal set spanning the full space
Standard 2D basis: e₁=[1,0], e₂=[0,1]
Any 2D point = a·e₁ + b·e₂

Why it matters for AI

A full-rank weight matrix has independent rows — it can transform data into any direction without losing information. A rank-deficient matrix collapses some dimensions → information loss.

PCA finds truly independent directions in your data. Feature engineering tries to remove redundant (dependent) features before training.

AI Application — Rank, Bottlenecks & LoRA

When a matrix has rank r, it only uses r independent directions — the rest are redundant. This is the mathematical basis for LoRA: researchers discovered that fine-tuning updates to large models have very low rank in practice. So instead of updating a full 4096×4096 matrix, they approximate it as A×B where A is 4096×4 and B is 4×4096 — using only 4 independent directions, not 4096.

Deep Dive — Independent Information vs. Redundancy

concept · why · how · example
1The Concept

A set of vectors {v₁, v₂, …, vₖ} is linearly independent if the only way to write the zero vector as c₁v₁ + c₂v₂ + … + cₖvₖ = 0 is with all cᵢ = 0. If any non-trivial combination produces zero, at least one vector is "redundant" — expressible as a combination of the others. The span of a set is the collection of all reachable points using linear combinations. A basis is a minimal set whose span is the whole space — like the smallest possible set of LEGO pieces from which you can build everything.

Think of independence as "non-redundant information." Three independent vectors in 3D give you full freedom to reach any point. Three dependent vectors might all lie in a 2D plane — no matter how you combine them, you can't escape that plane.
2Why It Matters in AI

The rank of a weight matrix equals the number of independent rows (or columns). A rank-deficient matrix is "wasteful" — it has more parameters than information capacity. Researchers found that fine-tuning updates to large LLMs are surprisingly low-rank: a 4096×4096 matrix update can often be approximated by an outer product of two skinny matrices (4096×r) × (r×4096) with r≈8. This is the foundation of LoRA, which makes fine-tuning GPT-scale models tractable on a single GPU.

3How It's Used — Step by Step
  1. Stack vectors as rows of a matrix M.
  2. Run row-reduction (Gaussian elimination) to triangular form.
  3. Count non-zero rows. That count is the rank — the number of truly independent directions.
  4. If rank < n, some vectors are redundant. In LoRA, this insight inspires factoring ΔW ≈ A · B where A is d×r and B is r×d, with r ≪ d.
4Worked Example

Finding the shadow of one vector onto another

☀️

Plain English: Imagine shining a light straight down onto a vector. The shadow it casts on the ground is its projection. We measure "how much of vector A points in the direction of vector B?" That shadow length is the projection.

Formula

// The vector component of a in direction b

// Just the length (signed number)

Geometric intuition

The projection of a onto b is the point on b's line that is closest to a. It's how much a "overlaps" with b. This is exactly what a neuron computes — how much does input x align with weight vector w?

AI Application — How a Neuron Fires

A single neuron computes output = w·x + b — this is a dot product, which is essentially projecting the input x onto the weight vector w. If the input aligns strongly with w (big projection), the neuron fires strongly. If perpendicular (no alignment), the neuron outputs ~0.

This is how neurons become feature detectors: a neuron whose weights form a pattern (like a horizontal edge) fires maximally when its input contains that exact pattern. The dot product measures how well the input matches the learned pattern.

Deep Dive — Decomposing One Vector Along Another

concept · why · how · example
1The Concept

Projecting a onto b means: of all the multiples of b, find the one that lies closest to a. The result is itself a vector along the b direction. The projection has two parts: a scalar projection (signed length: how far along b you go) and a vector projection (the actual vector pointing along b). The leftover piece a − proj_b(a) is perpendicular to b and is called the residual — it's everything in a that's NOT explained by b.

Projection is the simplest "explain a using b" operation. If b is a known direction, projection extracts how much of a lies along it. The residual is what's left unexplained — pure novelty.
2Why It Matters in AI

A single neuron is a projection: z = w·x measures how much input x aligns with the learned pattern w. Each filter in a CNN projects an image patch onto a learned feature direction. Least-squares regression is geometrically the projection of the target vector y onto the column space of the design matrix X. Even self-supervised learning often boils down to projecting representations and asking how much they agree.

3How It's Used — Step by Step
  1. Compute the dot product of the two vectors: a·b.
  2. Compute |b|² — the squared magnitude of the direction you're projecting onto.
  3. Divide to get the scalar projection coefficient: k = (a·b) / |b|².
  4. Multiply b by k to get the projection vector: proj_b(a) = k · b.
  5. (Optional) compute residual: r = a − proj_b(a). This is what b cannot explain.
4Worked Example

Project a = [3, 2] onto b = [4, 0] (the x-axis direction).

Deep Dive — Matrices as Functions

concept · why · how · example
1The Concept

A matrix is a 2D grid of scalars, but the useful way to think of it is as a function: an m×n matrix W maps an n-dimensional input vector to an m-dimensional output vector via y = Wx. Each row of W defines one output dimension as a linear combination of the input components. The shape (m × n) tells you the function's signature: it eats n-dim inputs and produces m-dim outputs. Transpose Wᵀ swaps rows and columns — the function flips direction (m-dim → n-dim).

Don't picture matrices as "tables of numbers." Picture them as boxes with arrows going in and out. W: ℝⁿ → ℝᵐ. The numbers inside W are just the wiring details that determine how the inputs get blended into outputs.
2Why It Matters in AI

A neural network is fundamentally a stack of matrices interleaved with nonlinear activations: x → W₁ → σ → W₂ → σ → … → ŷ. The matrices are everything that's learned — billions of weights are nothing but the numbers inside W₁, W₂, etc. GPT's "knowledge" lives in these matrices. Training adjusts them via gradient descent. So mastering matrices = mastering the substrate of modern AI.

3How It's Used — Step by Step
  1. Define the shape based on input/output dimensions. A layer mapping 768 → 3072 needs W of shape (3072 × 768).
  2. Initialize randomly (e.g., Xavier or He initialization with carefully chosen scale).
  3. Forward pass: for input x ∈ ℝ⁷⁶⁸, compute y = Wx + b ∈ ℝ³⁰⁷². This applies the linear transformation.
  4. Backward pass computes ∂L/∂W, an m×n gradient matrix.
  5. Update: W ← W − α·∂L/∂W — the matrix is itself updated by another matrix.

Deep Dive — The Single Most Expensive Operation in AI

concept · why · how · example
1The Concept

To multiply A (m×k) by B (k×n), the result C (m×n) has entry C[i][j] = Σ A[i][p] · B[p][j] over p = 1..k. Said differently: C[i][j] = (row i of A) · (column j of B). The "inner" dimensions must match (k = k); the "outer" dimensions become the result's shape (m × n). Matrix multiplication is not commutative — generally AB ≠ BA — and it's associative — (AB)C = A(BC), which lets us choose the cheapest order.

Think of A·B as composition of functions: first apply B (right-most acts first), then apply A. Two transformations stacked into one. Three layers of a neural network = one giant matrix product mathematically, though we don't fuse them in practice because of nonlinear activations between layers.
2Why It Matters in AI

Roughly 99% of GPU compute time in training and inference is matrix multiplication. The forward pass of GPT is a chain of matmuls; backpropagation is matmuls in reverse. Modern AI hardware (TPUs, NVIDIA Tensor Cores) is purpose-built to do nothing but matmuls efficiently. Even attention boils down to two big matrix products: QKᵀ and (softmax result)·V. When researchers talk about "FLOPs" for a model, they're really counting matmul operations.

3How It's Used — Step by Step
  1. Verify shapes match: A is (m×k), B must be (k×n). The shared k is the contraction dimension.
  2. For each output cell C[i][j]: grab row i of A and column j of B.
  3. Take their dot product — element-wise multiply, then sum.
  4. Place the scalar into C[i][j]. Repeat for all m×n cells.
  5. Total cost: m·n·k multiply-adds. A 4096×4096 by 4096×4096 product needs ~70 billion operations — done in milliseconds on a modern GPU.

1The Concept

2Why It Matters in AI

  4. Compose multiple transformations by multiplying matrices: C = B·A means "first apply A, then B."

Deep Dive — Determinant, Inverse, and Information Loss

concept · why · how · example
1The Concept

The determinant of a square matrix is a scalar that measures how the matrix scales volumes. In 2D, |det(W)| equals the area of the parallelogram formed by W's columns. Sign matters too: positive det preserves orientation, negative det flips it. If det = 0, the matrix collapses space into a lower dimension and cannot be inverted. The inverse A⁻¹ "undoes" A: A · A⁻¹ = I. Only square, full-rank matrices have inverses.

Determinant = "stretchiness factor." det=2 doubles all areas. det=0.5 halves them. det=0 squashes everything flat — and you can never un-squash, just like you can't recover the third dimension after photographing a 3D scene.
2Why It Matters in AI

Three uses dominate. First, checking invertibility: solving Ax = b directly only works when det(A) ≠ 0. Second, normalizing flows (a class of generative models) require computing log|det(Jacobian)| at every step to track how probability mass changes. Third, the singularity in the loss landscape — places where the Hessian has det = 0 — corresponds to flat minima and saddle points where optimization gets stuck.

3How It's Used — Step by Step
  1. For a 2×2 matrix A = [[a,b],[c,d]]: det = ad − bc. Quick rule: down-diagonal product minus up-diagonal product.
  2. If det ≠ 0, the inverse is A⁻¹ = (1/det) × [[d, −b], [−c, a]] — swap diagonal, negate off-diagonal, divide by det.
  3. Verify: compute A · A⁻¹ and confirm it equals the identity I.
  4. Solve Ax = b: apply A⁻¹ to both sides → x = A⁻¹ · b. (In practice, libraries solve via LU decomposition for stability — but the idea is the same.)

🧩

Plain English: "If I buy 2 apples and 3 bananas for $5, and 1 apple and 2 bananas for $3 — what does each fruit cost?" That's a system of linear equations. AI solves these (in millions of dimensions) to find optimal weights.

Methods

// Direct (small systems)
x = A⁻¹b  // if A is invertible

// Gaussian elimination
[A | b] → row reduce → solution

// Least squares (overdetermined)

Three possible outcomes

Unique solution — A is invertible (det≠0), exactly one x satisfies Ax=b. The system is perfectly determined.

No solution — overdetermined (more equations than unknowns). Use least squares — find x that gets as close as possible.

Infinite solutions — underdetermined (more unknowns than equations). Neural nets are always in this regime! Many weight combinations give zero training loss.

For small datasets, you can solve this exactly. For large datasets (n > 10,000), gradient descent is faster than computing the matrix inverse (which is O(n³)).

Deep Dive — Linear Regression as Linear Algebra

concept · why · how · example
1The Concept

The system Ax = b asks: "is there a vector x such that A applied to x produces exactly b?" Three cases. Square + full-rank A: unique solution x = A⁻¹b. Tall A (more equations than unknowns): usually no exact solution exists, but we can find the x that minimizes ||Ax − b||² — that's least squares. Wide A (more unknowns than equations): infinitely many solutions; we pick one with extra structure (e.g. minimum norm). Neural networks live in this third regime — vastly more parameters than training examples.

Geometrically, Ax travels through the column space of A — the set of all reachable outputs. If b is in that space, exact solution. If not, the best we can do is project b onto the column space and solve for that projection.
2Why It Matters in AI

Linear regression — the simplest ML model — is solved by the normal equations: w = (XᵀX)⁻¹Xᵀy. These come straight from least squares. Even when we use gradient descent for big problems, understanding the closed-form solution gives us the gold-standard answer to compare against. Beyond regression, every "physics-informed" or "constrained" optimization in AI ends up solving structured Ax = b systems many times.

3How It's Used — Step by Step (Least-Squares Linear Regression)
  1. Build the design matrix X — each row is a training example, each column a feature (add a column of 1s for the intercept).
  2. Stack targets y as a column vector.
  3. Compute XᵀX (small square matrix) and Xᵀy (column vector).
  4. Solve (XᵀX) · w = Xᵀy for w. In code: w = np.linalg.solve(X.T @ X, X.T @ y).
  5. Predict on a new x: ŷ = wᵀx.

Deep Dive — How Much Information Does a Matrix Really Hold?

concept · why · how · example
1The Concept

The rank of a matrix is the number of linearly independent rows (which always equals the number of independent columns). For an m×n matrix, rank ≤ min(m, n). The column space is the set of all outputs Ax can produce; its dimension equals the rank. The null space is the set of inputs that get sent to zero: {x : Ax = 0}. The famous rank-nullity theorem says: rank(A) + dim(null(A)) = n. Information either survives the matrix or gets destroyed — total dimensions accounted for.

Picture A as a funnel. Inputs in the null space drain out (lost forever). Whatever survives must lie in the column space. Rank tells you how wide the surviving channel is.
2Why It Matters in AI

Empirically, weight matrices in trained neural networks are often much lower rank than their nominal size. A 4096×4096 attention projection might effectively use only ~50 independent directions. This is the conceptual basis for LoRA (rank-r adapters), model distillation, and pruning. Conversely, encoder-decoder architectures intentionally create rank bottlenecks: the encoder squeezes rich data through a low-dimensional code, forcing the model to learn the most informative directions.

3How It's Used — Step by Step
  1. Run row reduction on A to row-echelon form.
  2. Count non-zero rows — that's the rank r.
  3. Find null space basis: back-substitute the reduced form, treat free variables as parameters; the resulting vectors span null(A).
  4. Verify rank-nullity: r + dim(null) should equal n.
  5. For low-rank approximation in practice, use SVD: keep top-r singular values, zero out the rest, reconstruct.

Deep Dive — A Matrix's Natural Axes

concept · why · how · example
1The Concept

An eigenvector of a square matrix A is a non-zero vector v that A only stretches: Av = λv. The scalar λ is its eigenvalue. Most vectors get rotated AND scaled when you apply A, but eigenvectors keep their direction — they're the matrix's "skeleton." Find them by solving det(A − λI) = 0 for λ (the characteristic polynomial), then plug each λ back to find the corresponding v from (A − λI)v = 0. A symmetric n×n matrix always has n real, mutually perpendicular eigenvectors.

Eigenvectors are A's "preferred directions." Apply A a thousand times to a random vector — it will progressively align with the eigenvector having the largest |λ|. This is the secret of power iteration, PageRank, and the spectral methods that dominate scientific computing.
2Why It Matters in AI

Eigenvectors of a covariance matrix are exactly the principal components in PCA — the directions of maximum variance in data. Eigenvalues of the loss Hessian tell you whether you're at a minimum (all positive), saddle (mixed signs), or maximum (all negative); they also bound the safe learning rate as α < 2 / λ_max. Eigenvalues of an LLM's weight matrices reveal how unstable training is: if λ_max grows much larger than 1, the network amplifies noise exponentially through layers.

3How It's Used — Step by Step
  1. Set up: compute A − λI (subtract λ from each diagonal entry).
  2. Take det(A − λI) — it's a polynomial in λ called the characteristic polynomial.
  3. Set it to zero and solve. The roots are the eigenvalues λ₁, λ₂, …
  4. For each λᵢ: solve (A − λᵢI) v = 0 for v. Any non-zero solution is an eigenvector.
  5. (Optional) Normalize each eigenvector to unit length for cleanliness.

Deep Dive — PCA from Scratch

concept · why · how · example
1The Concept

Principal Component Analysis is "finding the best photographic angle" for high-dimensional data. PCA finds an orthogonal set of directions (the principal components) such that the first PC captures the most variance, the second captures the most remaining variance perpendicular to PC1, and so on. Mathematically, the principal components are the eigenvectors of the data's covariance matrix, ordered by their eigenvalues (which equal the variance along each PC).

PCA assumes that the directions where your data spreads the most are the directions that carry the most information. Often that's true (a long, thin point cloud) but not always — sometimes important signal lives in low-variance directions.
2Why It Matters in AI

PCA delivers dimensionality reduction for free: keep just the top-k principal components and you've compressed your data while losing minimal variance. This makes downstream models faster and less prone to overfitting. PCA is also the standard preprocessing for visualizing high-D embeddings (compress 768D → 2D for plotting), for noise reduction (low-variance directions are often noise), and as a baseline against which neural compression methods like autoencoders are compared.

3How It's Used — Step by Step
  1. Center the data: subtract the mean of each column. Now the dataset has zero mean.
  2. Compute the covariance matrix: C = (1/(n−1)) · Xᵀ · X. This is d×d where d = number of features.
  3. Find eigenvectors and eigenvalues of C. Sort by eigenvalue, descending.
  4. Pick top-k eigenvectors as columns of a projection matrix W ∈ ℝᵈˣᵏ.

Deep Dive — SVD as Universal Decomposition

concept · why · how · example
1The Concept

Every real matrix A (m×n, any shape) can be written as A = U Σ Vᵀ where U is m×m orthogonal, V is n×n orthogonal, and Σ is m×n with non-negative singular values σ₁ ≥ σ₂ ≥ … ≥ 0 along the diagonal. Geometrically: any linear transformation = rotation, then axis-aligned stretch, then another rotation. The singular values measure how much each direction is stretched. Number of non-zero singular values equals the rank.

SVD is PCA's more general cousin — PCA only works on covariance matrices (square, symmetric, positive-semidefinite), but SVD works on any matrix. In fact, PCA is SVD applied to the centered data matrix.
2Why It Matters in AI

SVD provides the best rank-k approximation of any matrix (Eckart–Young theorem): truncate Σ to its top-k singular values, and you have the closest possible rank-k matrix to A measured in Frobenius or spectral norm. This drives compression: image compression (keep top singular values, discard the small ones), collaborative filtering (Netflix-style recommendations factor the user-item matrix as U Σ Vᵀ), and LoRA (re-parameterize ΔW as a rank-r product, motivated by the empirical low-rank-ness of fine-tuning updates).

3How It's Used — Step by Step
  1. Compute AᵀA (n×n) and find its eigenvectors v₁, …, vₙ — these become V's columns.
  2. Eigenvalues of AᵀA are σᵢ² — take square roots for singular values.
  3. Compute uᵢ = (1/σᵢ) · A · vᵢ for each non-zero σᵢ — these become U's columns.
  4. Assemble: A = U Σ Vᵀ. Numerically use np.linalg.svd(A).
  5. Truncate to rank k by keeping the first k columns of U, the first k rows of Vᵀ, and the top-k Σ entries: A_k = U[:,:k] · Σ[:k,:k] · Vᵀ[:k,:].

concept · why · how · example
1The Concept

Real data is multi-axis. An image is (channels × height × width). A video adds a time axis. A batch of sequences is (batch × tokens × embedding). Multi-head attention adds a heads axis. Deep Dive — Building a Clean Coordinate System

concept · why · how · example
1The Concept

Gram-Schmidt takes any set of linearly independent vectors and produces an orthonormal basis — vectors that are all unit length and mutually perpendicular. Process: keep the first vector, normalize it. For each next vector, subtract its projections onto every already-orthogonalized vector (so what remains is perpendicular to all of them), then normalize. The output: an orthonormal set spanning the same subspace as the input.

It's like building a coordinate system from scratch using whatever vectors you happen to have. The "subtract the projection" step removes overlap; the "normalize" step makes sure each axis is the same length.
2Why It Matters in AI

Orthogonal matrices are numerically stable — they don't amplify errors. QR decomposition (any matrix A = Q·R where Q is orthogonal) uses Gram-Schmidt under the hood and is preferred over normal-equations for solving least squares. Orthogonal initialization of weight matrices (specifically RNNs/Transformers) helps training stability. Multi-head attention works best when different heads learn (approximately) orthogonal features — orthogonalization regularizers explicitly enforce this.

3How It's Used — Step by Step
  1. Take v₁, the first input vector. Normalize: u₁ = v₁ / |v₁|.
  2. For v₂: compute its projection onto u₁: p = (v₂ · u₁) u₁. Subtract: w₂ = v₂ − p. Normalize: u₂ = w₂ / |w₂|.
  3. For v₃: subtract projections onto both u₁ and u₂: w₃ = v₃ − (v₃·u₁)u₁ − (v₃·u₂)u₂. Normalize.
  4. Continue for any remaining vectors. Result: an orthonormal basis {u₁, u₂, …}.

Deep Dive — The Compass That Drives All of Training

concept · why · how · example
1The Concept

The gradient of a scalar function L(w₁, w₂, …, wₙ) is the vector of all partial derivatives: ∇L = [∂L/∂w₁, ∂L/∂w₂, …, ∂L/∂wₙ]. It points in the direction of steepest ascent — the direction along which L grows fastest. The magnitude |∇L| tells you how steep the slope is. Critically, gradients are local: they only describe the immediate neighborhood, not far-away regions. The negative gradient, −∇L, is the direction of steepest descent — exactly what we need for minimization.

If the loss is a hilly landscape and you're standing at point w, the gradient is a tiny arrow at your feet pointing uphill. Walk against it = downhill. Take many small steps and (usually) you'll reach a valley.
2Why It Matters in AI

Training is just repeated gradient descent with the gradient telling each weight how to change. Without gradients, we'd have to try every possible weight combination — combinatorially impossible for billions of parameters. The chain rule (computed efficiently by backprop) gives us all the partials simultaneously in one backward pass, making training feasible. Modern variants like Adam, RMSProp, and SGD-momentum all use the gradient as input but adapt it (per-parameter scaling, momentum, decay) before applying.

3How It's Used — Step by Step
  1. Define a loss function L(w) that measures error and is differentiable in every weight.
  2. For each weight wᵢ, compute ∂L/∂wᵢ — the rate of change of L when wᵢ alone is nudged.
  3. Stack these partials into the gradient vector ∇L.
  4. Move in the negative direction: w ← w − α · ∇L. Loss decreases (locally).
  5. Stop conditions: when |∇L| ≈ 0 (flat region) or after a budget of steps.

Deep Dive — How Models Actually Learn

concept · why · how · example
1The Concept

Gradient descent is the iterative rule w ← w − α · ∇L(w). Each step nudges the weights opposite to the gradient direction. The learning rate α controls step size: too small → slow convergence; too large → overshoot or divergence. Stochastic gradient descent (SGD) uses one mini-batch's gradient instead of the full dataset's — noisy but vastly faster per step. Variants like Momentum, RMSProp, and Adam add running averages of past gradients to dampen oscillations and adapt α per-parameter.

2Why It Matters in AI

Every modern neural network is trained by gradient descent or one of its variants. Pretraining a 70B-parameter LLM is hundreds of thousands of gradient-descent steps on trillions of tokens. The choice of optimizer (SGD vs Adam vs AdamW), learning rate, schedule, and batch size are arguably the most consequential choices in any training run — get them wrong and the loss blows up or never decreases.

  1. Sample a mini-batch of training examples.

  3. Backward pass: compute gradient ∇L w.r.t. every weight.

  5. Update: for plain SGD, w ← w − α · ∇L. For Adam, scale by running averages of first and second moments first.
  6. Repeat for many epochs / steps. Track loss; check validation periodically.

Deep Dive — Chain Rule at Industrial Scale

concept · why · how · example
1The Concept

Backpropagation is just the chain rule applied to the computation graph of a model. For a composition L = f(g(h(w))), the chain rule says ∂L/∂w = ∂L/∂f · ∂f/∂g · ∂g/∂h · ∂h/∂w. The trick is doing this efficiently for billions of parameters: backprop walks the graph in reverse, multiplying gradients as it goes, reusing intermediate results so the cost is roughly the same as the forward pass — not exponential as it would be if computed naively.

Think of it as accountability. The loss is bad. Backprop walks back through the network asking each operation, "How much of this error is your fault?" — and assigns blame to its inputs and weights proportionally.
2Why It Matters in AI

Without backprop, training deep networks would be infeasible — the cost of computing each gradient one at a time scales with parameter count. Backprop computes all gradients in a single backward pass at the same cost as one forward pass. Every framework (PyTorch, TensorFlow, JAX) implements it as reverse-mode automatic differentiation: record every operation in a graph, then walk backward applying transposed Jacobians.

3How It's Used — Step by Step
  1. Forward pass: compute and cache every intermediate (z₁ = W₁x, a₁ = σ(z₁), z₂ = W₂a₁, …) until reaching the loss L.
  2. Initialize the upstream gradient at the loss: δ = ∂L/∂L = 1.
  3. Walk backward. At each operation, multiply δ by the local Jacobian to get the gradient w.r.t. that operation's inputs.
  4. For weights at each linear layer: ∂L/∂W = δ · xᵀ (outer product). Save this for the optimizer.
  5. For inputs at each linear layer: ∂L/∂x = Wᵀ · δ. Pass this further back as the new δ.
  6. Optimizer step uses the accumulated weight gradients: W ← W − α · ∂L/∂W.

concept · why · how · example
1The Concept

Each token in the sequence produces three vectors via three learned projection matrices: a Query Q, a Key K, and a Value V. The token "asks" what it's looking for via Q. Use those weights to mix the V vectors → the new representation. Formula: Attention(Q, K, V) = softmax(QKᵀ / √d) · V.

Q = "what am I looking for?" · K = "what am I about?" · V = "what info do I provide?" The Q·K dot product is similarity; softmax turns it into a probability distribution over tokens; the weighted sum of V's is the output. Three matrix multiplies + a softmax = the most important operation in modern AI.
2Why It Matters in AI

Attention solved the long-standing problem of long-range dependencies in sequences — RNNs forgot, LSTMs were slow, but attention lets every token see every other token in O(1) hops. This unlocked GPT, BERT, and the entire LLM revolution. Vision Transformers apply the same operation to image patches. Diffusion models use cross-attention between image patches and text tokens. Even DNA-folding (AlphaFold) uses attention. The √d divisor keeps softmax in a sensitive regime; multi-head attention runs the operation in parallel with different Q/K/V projections to capture different relationships simultaneously.

3How It's Used — Step by Step
  1. Project the input embedding matrix X into Q, K, V via three learned weight matrices: Q = X·W_Q, K = X·W_K, V = X·W_V.
  2. Compute scores S = Q · Kᵀ — a matrix where S[i][j] is "how much should token i pay attention to token j?"
  3. Scale by 1/√d to keep softmax stable: S = S / √d_k.
  4. Softmax over each row of S so weights sum to 1: A = softmax(S).
  5. Weight the values: output = A · V. Each row of output is a weighted sum of V's, with the attention pattern of the corresponding query.
  6. (Multi-head) repeat steps 1–5 in parallel with different W_Q/K/V's, then concatenate outputs and project.

Deep Dive — Softmax

concept · why · how · example
1The Concept

Softmax takes a vector of arbitrary real numbers (logits) and converts them into a probability distribution: every output is between 0 and 1, and they all sum to exactly 1. The formula is softmax(x)ᵢ = exp(xᵢ/T) / Σⱼ exp(xⱼ/T). The exponential ensures positivity and amplifies differences — a logit that is just slightly higher than the others ends up with a disproportionately large share of the probability. The temperature T scales the logits before exponentiation: T < 1 sharpens (winner takes more), T > 1 flattens (more uniform).

💡 Think of softmax as a "soft" version of argmax. Argmax says "the biggest one wins everything"; softmax says "the biggest one wins most, but the others get a slice proportional to how close they were."
2Why It Matters in AI

Almost every neural network classifier ends with softmax — it converts raw scores into something a loss function (cross-entropy) can compare to a true label. Inside transformers, softmax is what turns attention scores into weights that sum to 1, so each token's new representation is a proper weighted average of others. Softmax is also fully differentiable, so gradients flow through it cleanly during backprop. Without softmax there is no probabilistic LLM output, no attention mechanism, and no standard classification head.

3How It's Used — Step by Step
  1. Get logits from the final linear layer (one number per class or vocabulary token).
  2. Subtract the max from every logit. This is a numerical-stability trick — it prevents exp() from overflowing on large positive numbers — and gives identical results because the constant cancels out in the ratio.
  3. Divide by temperature T if you want to control sharpness (T=1 leaves it alone).
  4. Exponentiate each shifted logit.
  5. Sum all the exponentials.
  6. Divide each exponential by that sum — those are your probabilities.
  7. Sample from the distribution (LLM generation) or take argmax (classification).

Deep Dive — Convolution

concept · why · how · example
1The Concept

A convolution slides a small grid of weights — the kernel (also called a filter) — across a larger input grid. At every position, it computes a dot product between the kernel and the patch of input it currently overlaps, and writes that single number into the output. The kernel is the same at every position: this is parameter sharing. A 3×3 kernel has only 9 weights, yet it produces a feature map covering the whole image. Different kernels detect different patterns: one might fire on vertical edges, another on horizontal edges, another on a specific colour blob.

💡 A convolution is a dot product, repeated. The kernel is a tiny "template," and at each position the output answers the question: how much does this patch of the image look like the template?
2Why It Matters in AI

Convolutions encode three powerful priors that match how images actually work: locality (nearby pixels are related), translation invariance (a cat is a cat whether it's in the top-left or bottom-right), and parameter sharing (one edge detector works everywhere). A fully connected layer on a 224×224 RGB image would need ~150K weights per neuron; a 3×3 conv needs 27. That parameter efficiency is why CNNs dominated computer vision for a decade and still power the vision side of multimodal models. Vision transformers eventually replaced them in many tasks, but they internally cut the image into patches — a discrete echo of the same locality prior.

3How It's Used — Step by Step
  1. Choose kernel size (3×3 is most common) and the number of kernels in the layer (e.g. 64).
  2. Position the kernel at the top-left of the input.
  3. Element-wise multiply the kernel with the input patch underneath, then sum. Write the result in the output map.
  4. Slide the kernel by the stride (usually 1) and repeat until the whole image is covered.
  5. Apply activation (typically ReLU) to the output map.
  6. Stack multiple kernels — each produces its own output map (channel). 64 kernels → 64 output maps.
  7. im2col trick: in practice GPUs flatten every patch into a row of a big matrix and do one giant matrix multiply with the stacked kernels. Convolution is fundamentally a matmul in disguise.

Deep Dive — Embeddings

concept · why · how · example
1The Concept

An embedding is a dense vector — usually 128 to 4096 numbers — that represents the meaning of something (a word, a sentence, an image, a user, a product) as a point in a high-dimensional space. The geometry is the meaning: items with similar meaning sit close together, dissimilar items sit far apart. Each dimension doesn't have to mean anything specific by itself — meaning emerges from relative positions. That's why "king − man + woman ≈ queen" works in a well-trained word embedding space.

💡 An embedding is meaning-as-coordinates. Once your concepts live in vector space, "what's similar to X?" becomes "what vectors are nearest to X?" — a problem your laptop can solve millions of times per second.
2Why It Matters in AI

Embeddings are the universal interface of modern AI. Search, retrieval-augmented generation (RAG), recommendations, deduplication, clustering, anomaly detection, and multimodal models like CLIP all reduce to the same recipe: encode everything as vectors, then use distance or dot product to compare them. They turn fuzzy human concepts ("documents about climate policy", "users who liked this") into operations a GPU can do at scale: a single matrix multiplication can compare one query against millions of items in milliseconds.

3How It's Used — Step by Step
  1. Train an encoder — Word2Vec, BERT, sentence-transformers, CLIP — to map inputs into vectors using contrastive or self-supervised objectives that pull similar pairs together and push dissimilar ones apart.
  2. Encode your corpus: run every document/image/product through the encoder once and store the resulting vectors in a vector database (FAISS, Pinecone, pgvector).
  3. Encode the query at runtime using the same encoder, producing one query vector.
  4. Compute similarity between the query vector and every stored vector — typically cosine similarity (dot product of normalised vectors).
  5. Take the top-k by similarity score. These are the most relevant items.
  6. Use the results: as RAG context for an LLM, as recommendations, as search results, as nearest-neighbour classifications.
4Worked Example

Tiny 3-D embeddings to make the geometry visible. We'll measure how similar "cat" is to "kitten" versus "car" using cosine similarity.

Cosine similarity in toy embedding space
Givencat = [0.9, 0.1, 0.2],   kitten = [0.85, 0.15, 0.18],   car = [0.1, 0.8, 0.7]
Step 1Norms: ||cat|| = √(0.81+0.01+0.04) = 0.927,   ||kitten|| = √0.7774 = 0.882,   ||car|| = √1.14 = 1.068
Step 2cat · kitten = 0.9·0.85 + 0.1·0.15 + 0.2·0.18 = 0.765 + 0.015 + 0.036 = 0.816
Step 3cos(cat, kitten) = 0.816 / (0.927 · 0.882) = 0.816 / 0.818 ≈ 0.997
Step 4cat · car = 0.9·0.1 + 0.1·0.8 + 0.2·0.7 = 0.09 + 0.08 + 0.14 = 0.31
Step 5cos(cat, car) = 0.31 / (0.927 · 1.068) = 0.31 / 0.990 ≈ 0.313

A real RAG system does exactly this calculation, just with 1536-dimensional vectors and millions of documents. The encoder is what does the heavy lifting; once it's trained, retrieval is just a fast matrix multiplication followed by a top-k.
