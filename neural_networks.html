What Is a Neural Network?

A flexible function that learns from data — fit a curve through noise
🧩

Plain English: A neural network is just a function — input goes in one side, output comes out the other. The magic: instead of you writing the rules (if-then-else), the network discovers the rule by adjusting thousands of tiny knobs (weights) until its output matches the right answer on examples you show it. Show it photos of cats and dogs, and it learns to map pixels → label. Show it stock prices, and it learns to map yesterday → tomorrow.

AI Deep Dive — Neural Networks

Topic 01
📚 The Concept

A neural network is a parametric function NN(x; θ): a chain of matrix multiplies and nonlinearities, where θ are learnable parameters (weights and biases). The Universal Approximation Theorem says: with enough hidden units, even a single-hidden-layer network can approximate any continuous function to arbitrary accuracy. In practice, depth (more layers) is exponentially more efficient than width — that's why we go deep.

💡 Intuition: Imagine a Lego sculptor with millions of bricks. Each brick (weight) is tiny, but stacked just right they form any shape — a face, a city, a galaxy. Training is just slowly nudging each brick until the sculpture matches the target.
❓ Why AI needs this

Every modern AI system is a neural network underneath. The framework is universal:

  • Image recognition: pixels → object class (CNN)
  • Translation: English tokens → French tokens (Transformer)
  • ChatGPT: previous tokens → next token (Transformer LLM)
  • AlphaGo: board position → move probabilities (CNN + tree search)
  • Self-driving: camera + lidar → steering angle (CNN/Transformer hybrid)
⚙️ How AI uses it

Every NN follows the same recipe:

AI Deep Dive — The Neuron

Topic 02
📚 The Concept

A single neuron computes a weighted sum z = w·x + b, then applies a nonlinear activation a = σ(z). The weights w are a vector of importance scores; the bias b is a learnable offset. Geometrically, w·x + b = 0 defines a hyperplane in input space — everything on one side activates, everything on the other doesn't. So one neuron = one half-space classifier. That's not much, but combine many neurons and you can chop up the input space into arbitrary regions.

💡 Intuition: A neuron is a doorman with a clipboard. Each input is a person showing up; the weight is how much the doorman cares about that feature ("are they on the list?", "do they look the part?"). Sum the scores, add a baseline mood (bias), and decide: open the door or not.
❓ Why AI needs this

The neuron is the universal building block — every architecture is just billions of these arranged differently:

  • Dense / fully-connected: every input → every neuron
  • Convolutional: neurons share weights across spatial positions
  • Recurrent: neurons feed back into themselves over time
  • Attention: neurons compute similarity scores and route info
  • GPT-3 has ~175 billion weights — but the basic unit is still w·x + b
⚙️ How AI uses it

One linear layer = many neurons in parallel:

# 1 neuron with 3 inputs: w = torch.randn(3); b = torch.randn(1) z = (w * x).sum() + b # weighted sum a = F.relu(z) # activation # 128 neurons in parallel = a "Linear" layer: layer = nn.Linear(in_features=3, out_features=128) # stores W: (128, 3), b: (128,) out = layer(x) # shape (128,)

The nonlinearity — without it, deep networks collapse to a single line
⚡

AI Deep Dive — Activations

Topic 03
📚 The Concept

An activation function is a fixed nonlinear transformation applied element-wise to a layer's outputs. Without it, a stack of N linear layers W₁(W₂(W₃·x)) collapses to a single linear map (W₁W₂W₃)·x — no expressive gain from depth. The nonlinearity is what lets a deep network carve curved decision boundaries, learn XOR, and ultimately approximate any continuous function. The choice of activation matters because gradients flow back through it: saturating activations (sigmoid, tanh) cause vanishing gradients in deep networks, while ReLU avoids this for positive inputs but causes the "dying ReLU" problem.

💡 Intuition: Without activations, your network is one giant linear regression. ReLU folds the input space along a line. Stack many of them and you fold reality into ever finer creases — that's how a network with 10 layers can isolate a cat in pixel space.
❓ Why AI needs this

Different architectures use different activations for principled reasons:

  • ReLU dominates ConvNets and most MLP hidden layers (cheap, no saturation)
  • Sigmoid is used at the output for binary classification (it's a probability)
  • Softmax (multi-class sigmoid) is used at the output of LLMs and classifiers
  • GELU powers transformers (BERT, GPT-2, GPT-3) — smoother than ReLU near 0
  • SwiGLU (GPT-3.5+, Llama, Gemma) is a gated variant — even better in big models
⚙️ How AI uses it

Activations are usually a single line:

x = F.relu(linear1(x)) # hidden layers in CNN/MLP x = F.gelu(linear1(x)) # transformers x = F.softmax(logits, dim=-1) # final layer for classification x = torch.sigmoid(score) # binary head, e.g. spam yes/no

AI Deep Dive — Forward Pass

Topic 04
📚 The Concept

The forward pass is a sequence of affine transforms (W·x + b) interleaved with nonlinearities. With L layers, the network computes h_l = σ(W_l · h_{l-1} + b_l) where h_0 = x. Each layer's matrix W_l reshapes its input, and the activation σ keeps things nonlinear. The shapes of the W matrices determine the architecture: input_dim → hidden_dim → ... → output_dim. The forward pass is purely computational — no learning happens. It just predicts.

💡 Intuition: Forward pass is an assembly line. Raw materials (pixels, words, numbers) enter at one end. Each station (layer) does one specific transformation. Out comes a finished product (a class probability, a translation, a stock prediction).
❓ Why AI needs this

Forward pass is what runs at inference time — every time you query ChatGPT or run face recognition:

  • Image classifier: 224×224×3 pixels → 1000 class probabilities (~25 layers in ResNet)
  • LLM forward pass: tokens → next-token probabilities (~96 layers in GPT-3)
  • Forward pass cost = inference latency. Optimising it is most of "MLOps"
  • Batch the forward pass — run many examples through the same matrices simultaneously
⚙️ How AI uses it

The forward pass is the __call__ of every model:

class MLP(nn.Module): def __init__(self): self.l1 = nn.Linear(784, 128) self.l2 = nn.Linear(128, 64) self.l3 = nn.Linear(64, 10) def forward(self, x): # ← forward pass h1 = F.relu(self.l1(x)) h2 = F.relu(self.l2(h1)) return self.l3(h2) # logits logits = model(x) # invokes forward()

AI Deep Dive — Multi-Layer Networks

Topic 05
📚 The Concept

A multi-layer perceptron (MLP) with L hidden layers can carve input space into up to 2^N distinct linear regions, where N is the total number of ReLU units — and crucially, depth makes this efficient. A theorem by Montúfar et al. (2014) shows that a deep ReLU network with k layers and n units per layer creates regions that grow like Ω((n/d)^((k-1)d) · n^d), versus only O(n^d) for a single-layer network with the same units. Translation: doubling depth roughly squares your expressive power — that's why 96-layer transformers work where 1-layer networks can't.

💡 Intuition: Wide-but-shallow networks try to memorise every input region separately. Deep networks build a hierarchy: edges → textures → parts → objects. The same edge-detector neuron is reused for cats, cars, and clouds — that's compositional efficiency.
❓ Why AI needs this

Depth is what makes modern AI work:

  • AlexNet (2012, 8 layers) sparked the deep learning revolution
  • ResNet (2015, 152 layers) cracked ImageNet thanks to skip connections
  • GPT-3 (2020) has 96 transformer layers — each one composes the previous
  • Compositionality: layer 1 sees pixels, layer 50 sees concepts
  • Skip / residual connections let gradients flow through 100+ layers without dying
⚙️ How AI uses it

Build a deep MLP in 4 lines:

model = nn.Sequential( nn.Linear(input_dim, 256), nn.ReLU(), nn.Linear(256, 256), nn.ReLU(), nn.Linear(256, 256), nn.ReLU(), nn.Linear(256, num_classes)) # For really deep nets (100+ layers): use residual blocks class ResBlock(nn.Module): def forward(self, x): return x + self.layers(x) # ← skip connection

A single number measuring how wrong the network is — what training minimises
📏

Plain English: The loss is a "wrongness score." Predict 0.9 when the truth is 1.0? Loss is small. Predict 0.1 when the truth is 1.0? Loss is huge. Training's only job is to make this number go down. Different problems use different loss functions: MSE for regression (predicting house prices), cross-entropy for classification (cat vs dog), contrastive for embeddings, etc.

AI Deep Dive — Loss Functions

Topic 06
📚 The Concept

A loss function L(ŷ, y) takes the network's prediction ŷ and the true target y and outputs a non-negative scalar — zero meaning "perfect", larger meaning "more wrong". Training minimises L averaged over the dataset. The shape of L matters because training only sees its gradient: a flat loss gives no learning signal; a sharp loss gives strong signal. Cross-entropy is the workhorse for classification because of one beautiful property — minimising CE between the data distribution P and the model distribution Q is equivalent to maximum likelihood estimation of Q's parameters.

💡 Intuition: Loss is the network's "pain signal." MSE says "I'm slightly wrong, here's a gentle nudge." Cross-entropy says "I was 99% sure of the wrong answer — punish me severely so I never do it again." Choosing the loss is choosing what kind of mistakes you most want to fix.
❓ Why AI needs this

Loss choice defines the problem:

  • MSE for regression — house prices, stock prices, height predictions
  • BCE for binary — spam vs ham, cancer vs benign, click vs no-click
  • Cross-entropy for multi-class — ImageNet (1000 classes), next-token in LLMs
  • Contrastive (CLIP, SimCLR) for similarity learning
  • Triplet loss for face recognition (anchor + positive vs negative)
  • Custom losses for ranking (LambdaRank), GANs (Wasserstein), RL (PPO clip)
⚙️ How AI uses it

Each task has a one-line standard loss:

loss = F.mse_loss(pred, target) # regression loss = F.binary_cross_entropy(pred, target) loss = F.cross_entropy(logits, labels) # multiclass — combines log+softmax loss = F.huber_loss(pred, target) # robust to outliers loss = -(target * torch.log(pred+1e-9)).sum() # custom

AI Deep Dive — Gradient Descent

Topic 07
📚 The Concept

Gradient descent updates each parameter θ in the direction that locally reduces the loss the fastest: θ ← θ − α∇L. The gradient ∇L is a vector pointing uphill in loss space; we move opposite. The learning rate α controls step size: too small → glacially slow; too large → overshoots, can diverge. Stochastic Gradient Descent (SGD) approximates the true gradient using a small mini-batch of examples — noisy but fast, and the noise actually helps escape sharp minima. The condition number of the Hessian (ratio of largest to smallest curvature) determines how badly gradient descent zigzags in narrow valleys.

💡 Intuition: The gradient is your compass — it always points to the steepest path. The learning rate is your stride length. Take strides too short and you'll never reach the bottom. Stride too long and you overshoot and bounce around. The art of training is finding the right stride for your terrain.
❓ Why AI needs this

Gradient descent is the only algorithm that scales to billions of parameters:

  • Closed-form solutions (linear regression normal equations) are O(n³) — impossible at scale
  • Random search and genetic algorithms work poorly with millions of parameters
  • Gradient descent is O(parameters) per step — scales linearly
  • Combined with backprop, you get cheap gradients even for 100B+ parameter models
  • Mini-batch SGD also has implicit regularising effect (noise → flatter minima)
⚙️ How AI uses it

One line of PyTorch is a complete gradient descent step:

optimizer = torch.optim.SGD(model.parameters(), lr=0.01) for batch_x, batch_y in dataloader: loss = criterion(model(batch_x), batch_y) optimizer.zero_grad() # clear old gradients loss.backward() # compute ∇L via backprop optimizer.step() # θ ← θ − α∇L

Compute gradients efficiently via the chain rule — once per training step, not per parameter
⬅️

Plain English: The naive way to compute gradients: nudge each weight by ε, see how loss changes, repeat for all 175 billion weights. That'd take forever. Backprop is the trick that computes all gradients in a single backward pass, leveraging the chain rule. It's like calculating how much each ingredient affected the final cake by tracing the recipe in reverse — finished cake → batter → mix → individual ingredients.

AI Deep Dive — Backpropagation

Topic 08
📚 The Concept

Backpropagation is the application of the chain rule to a computation graph. Forward pass: compute activations layer-by-layer and cache them. Backward pass: starting from ∂L/∂ŷ = 1, propagate gradients backward, multiplying by local derivatives at each node. Crucially, this gives every parameter's gradient in one backward sweep — same total cost as the forward pass (≈ O(parameters)). Without backprop, training would cost O(parameters²); with it, O(parameters). That's the algorithm that makes deep learning practical.

💡 Intuition: Forward pass = "how did we get this prediction?" Backward pass = "who's responsible for the error, and by how much?" Backprop assigns blame proportionally — weights that contributed most to the wrong answer get the biggest correction.
❓ Why AI needs this

Without backprop, modern deep learning is impossible:

  • GPT-4 has ~1.7 trillion parameters — naive gradients would take centuries

AI Deep Dive — Initialization

Topic 10
📚 The Concept

The principle behind Xavier (Glorot & Bengio, 2010) is simple: keep the variance of activations roughly constant across layers in the forward pass, and the variance of gradients roughly constant in the backward pass. For a layer with n_in inputs, this means weights drawn from N(0, 1/n_in). He initialization (He et al., 2015) accounts for ReLU killing half the inputs — the variance needs to be doubled, so W ~ N(0, 2/n_in). Without proper init, deep networks suffer from vanishing or exploding activations and gradients — both stop learning. Modern architectures (transformers, ResNets) often combine smart init with normalisation layers (LayerNorm, BatchNorm) for additional robustness.

💡 Intuition: A signal travelling through 50 layers without proper init is like a phone call routed through 50 noisy amplifiers — by the time it reaches the destination, it's either silent (vanished) or screeching feedback (exploded). The right init keeps gain ≈ 1 at every stage.
❓ Why AI needs this

Bad init was a top reason deep nets didn't work pre-2010:

  • 2006-2010: deep nets needed unsupervised pretraining (autoencoders) just to initialise
  • 2010 Xavier paper: showed math-based init works without pretraining
  • 2015 He paper: extended to ReLU networks, enabling 100+ layer ResNets
  • Transformers: use smaller init (0.02 standard deviation) + LayerNorm for stability
  • Modern frameworks default to Kaiming/He init for nn.Linear and nn.Conv2d
⚙️ How AI uses it

AI Deep Dive — Regularization

Topic 11
📚 The Concept

Regularization restricts the hypothesis space the optimiser can settle in. Adding λ·||w||² to the loss is mathematically equivalent to placing a Gaussian prior N(0, 1/λ) on the weights and doing maximum-a-posteriori (MAP) estimation — the network is now biased toward small weights unless data forces otherwise. L1 (Laplace prior) creates exact zeros: parameters whose contribution is small enough get pushed to zero, automatically performing feature selection. Weight decay in optimisers like AdamW is L2 implemented in the update rule directly (rather than via the loss): w ← w − α(∇L + λw) = w(1 − αλ) − α∇L.

💡 Intuition: Without regularisation, the network is told only "match the training data perfectly." It learns elaborate ways to do so. Add λ·||w||² and you also tell it: "and please do it as quietly as possible." The network learns the simplest function that fits — and simple functions generalise better.
❓ Why AI needs this

L2 / weight decay is the most universal regulariser in DL:

  • Used in virtually every model — λ between 1e-5 and 1e-2 typical
  • L1 used when sparsity matters (interpretable models, embedded systems)
  • AdamW in transformers — decoupled weight decay (separate from gradient adaptation)
  • Often combined with dropout, data augmentation, and early stopping for best results
  • Lasso (L1) is the go-to feature-selection method in classical ML
⚙️ How AI uses it

AI Deep Dive — Dropout

Topic 12
📚 The Concept

Dropout is, mathematically, an approximation to training an exponential ensemble of networks: every step, a random sub-network is sampled and trained. At test time, using all neurons is approximately equivalent to averaging predictions across all 2^N possible sub-networks. The intuition: a neuron can't rely on a specific other neuron being there (it might be dropped), so it must encode useful features that survive the loss of any neighbour. This forces redundant, robust representations. Inverted dropout scales surviving activations by 1/(1−p) during training so test-time outputs need no rescaling.

💡 Intuition: Imagine training a basketball team where any 5 players might be benched on any given play. Every player must be capable of every role; no one becomes indispensable. The team learns redundant strategies — and at game time, when all 12 are available, they're a far stronger collective.
❓ Why AI needs this

Dropout is one of the most cost-effective regularisers ever invented:

  • AlexNet (2012) used dropout 0.5 on FC layers — was a major reason it beat hand-crafted features
  • Standard in transformers — applied after attention and after FFN, p = 0.1 typical
  • Particularly powerful for small datasets where overfitting is the main risk
  • Works with any architecture: MLPs, CNNs, RNNs, Transformers
  • Variants: SpatialDropout (drop whole channels in CNNs), DropPath (drop residual blocks)
⚙️ How AI uses it

Dropout is a layer; framework handles train/eval automatically:

AI Deep Dive — Batch Normalization

Topic 13
📚 The Concept

BatchNorm computes statistics across the batch dimension: for each channel, take the mean and variance over the B samples in the mini-batch, normalise, then apply learnable scale γ and shift β. At inference time, it uses running averages of μ and σ² collected during training (since you may have a single example, not a batch). The original 2015 paper claimed BN reduces "internal covariate shift"; later work (Santurkar 2018) showed the real benefit is smoothing the loss landscape — making it less sensitive to choice of learning rate and init. LayerNorm is the cousin used in transformers — it normalises across feature dimensions per-sample (no batch dependence, works for any sequence length).

💡 Intuition: Without BN, every weight update sends a shockwave through the deeper layers — each one suddenly sees a totally different input distribution and has to adapt. BN puts each layer in a well-calibrated input regime, no matter what the previous layers do. The deeper layers can focus on learning, not adapting.
❓ Why AI needs this

BN was the breakthrough that made training really deep CNNs practical:

  • Pre-2015: training a 30-layer net was a nightmare of careful init

  • BN typically lets you increase learning rate 10×, train 5× faster
  • Transformers use LayerNorm instead (sequences vary in length, can't batch-normalise)
  • Other variants: GroupNorm (small batches), InstanceNorm (style transfer), RMSNorm (LLMs)

📚 The Concept

The learning rate is the single most important hyperparameter in deep learning. Too high → divergence; too low → glacial training. AI Deep Dive — Overfitting & Generalization

Topic 15
📚 The Concept

The classical bias-variance tradeoff: simple models have high bias (consistently miss the truth) but low variance (predictions are stable across different training sets). Complex models have low bias but high variance (memorise quirks of the training set). Total expected test error = Bias² + Variance + irreducible noise. The remedies for high variance: more data (variance shrinks ∝ 1/√n), regularisation (constrains the hypothesis class), and ensembling. Strikingly, modern deep learning seems to defy this: massively over-parameterised networks (more weights than data) often generalise well — the "double descent" phenomenon. The leading theory: SGD has an implicit bias toward flat, low-norm solutions that generalise.

💡 Intuition: Memorising is easy when you have few examples and a big brain. Real learning is harder. Modern AI works partly because the data is so vast that even huge models can't memorise it all — they're forced to find the pattern.
❓ Why AI needs this

Detecting and preventing overfitting is the daily job of an ML engineer:

  • Always split data into train / validation / test (typical: 80/10/10 or 90/5/5)
  • Watch validation loss every epoch — if it starts rising, you're overfitting
  • Early stopping: snapshot the model at lowest val loss, ignore later epochs
  • More data is the most reliable cure — followed by augmentation if data is scarce
  • Modern deep learning achieves "benign overfitting" — but only with enough data and proper regularisation
⚙️ How AI uses it

Standard pipeline tracks train and val loss:

AI Deep Dive — Convolutional Networks

Topic 16
📚 The Concept

A 2D convolutional layer takes a 3D input (channels × height × width) and applies multiple filters. Each filter is a small 3D tensor (channels × k × k); it slides over the spatial dimensions, computing dot products and producing one output channel. Three properties make this powerful: local connectivity (each output depends only on a small input region — captures local patterns), weight sharing (same kernel everywhere — translation equivariance), and parameter efficiency (k²·C parameters per filter regardless of image size). Stack convs with pooling (downsampling) and you build a hierarchy: layer 1 finds edges, layer 5 finds textures, layer 20 finds object parts, final layer classifies.

💡 Intuition: A CNN is like a search team with a small set of magnifying glasses. Each glass spots a specific pattern — an edge, a corner, an eye. The team sweeps each glass over every spot in the image and notes where each pattern shows up. Higher layers combine these notes into "this region looks like a face."

  • EfficientNet (2019) — compound scaling, efficient compute usage
  • Used in medical imaging, satellite imagery, self-driving cars, video understanding
  • Vision Transformers (2020+) compete with CNNs but at much higher cost — CNNs still dominate efficiency-bound deployments

AI Deep Dive — RNNs & LSTMs

Topic 17
📚 The Concept

An RNN processes a sequence x₁, x₂, …, x_T by maintaining a hidden state h_t that summarises everything seen so far. The recurrence h_t = f(x_t, h_{t-1}) means information can flow forward through time — but in vanilla RNNs, gradients during backprop-through-time get multiplied by Jacobians of f at every step, so they either vanish (norm < 1) or explode (norm > 1) over long sequences. LSTMs (Hochreiter & Schmidhuber, 1997) solve this by introducing a cell state c_t that is modified additively via gates: a forget gate decides what to erase, an input gate decides what to add, an output gate decides what to expose to the next layer. Gradients flow back through the additive cell-state path nearly unchanged — the network can remember things hundreds of steps in the past. GRUs simplify the gating with just two gates and similar performance.

💡 Intuition: A vanilla RNN is like a goldfish — it has memory, but only of the last few seconds. An LSTM is a goldfish with a notebook: it can decide what to write down, what to erase, and what to read out. The notebook persists; only entries you explicitly forget are lost.
❓ Why AI needs this

RNNs / LSTMs powered NLP from 2014 to 2018, before transformers took over:

  • Seq2seq translation (2014) — encoder-decoder LSTMs translated languages
  • Speech recognition (Siri, Google Assistant pre-2018) — bidirectional LSTMs
  • Text generation, music generation, video captioning — all LSTM-based pre-2018
  • Now mostly replaced by transformers in NLP, but still used for: time series forecasting, on-device speech, low-latency streaming
  • Mamba/State Space Models (2023+) — modern resurgence of "RNN-like" architectures with linear-time inference
⚙️ How AI uses it

AI Deep Dive — Embeddings

Topic 18
📚 The Concept

An embedding is a learned lookup table: a matrix E of shape (V, d) where V is vocabulary size and d is embedding dimension. Each row is a vector representation of one token. Why dense vectors and not one-hot? Generalisation: if "dog" and "cat" have similar vectors, a downstream task that worked for one can transfer to the other. Compositionality: vector arithmetic encodes relations (capital_of, gender, plural). Embeddings are trained either jointly with a downstream task (end-to-end) or via self-supervised objectives like Word2Vec, GloVe, or modern contrastive methods (CLIP, SimCSE). In transformers, embeddings are followed by attention layers that further refine token representations contextually.

💡 Intuition: One-hot vectors are like postal codes — they uniquely identify but tell you nothing about the place. Embeddings are like GPS coordinates — distances are meaningful, neighbours are geographically close, and you can do directional reasoning ("go 100m north of Paris → suburbs").
❓ Why AI needs this

  • Modern: token embeddings inside BERT, GPT, T5, Llama — learned end-to-end
  • CLIP (2021) embeds text and images into the SAME 512-d space — basis of DALL-E, Stable Diffusion
  • Recommender systems use embeddings for users + items (Spotify, Netflix, YouTube)
  • RAG (retrieval-augmented generation) uses embeddings to find relevant docs by cosine similarity
  • Vector databases (Pinecone, Qdrant, FAISS) index millions of embeddings for fast nearest-neighbour search

AI Deep Dive — Attention

Topic 19
📚 The Concept

Self-attention computes, for each position, a weighted average over all positions. Each token x_i is projected into three vectors: a query q_i (what am I looking for?), a key k_i (what do I offer?), and a value v_i (what info do I carry?). The score s_ij = q_i · k_j / √d measures how much token i wants info from token j. Softmax over j gives attention weights. The output for position i is Σⱼ a_ij · v_j — a content-routed mixture. The √d scaling keeps the softmax in a regime where gradients flow well; without it, large d causes attention weights to become near-one-hot and gradient vanishes. Multi-head attention runs several attention computations in parallel with different W_Q, W_K, W_V — each head can specialise on different relations (syntactic, semantic, positional).

💡 Intuition: Imagine every word in a sentence sending out a "what am I about?" signal (query) and answering everyone else's signal with a "here's what I am" reply (key + value). Each word listens hardest to the words whose replies match its question best. By stacking 12 or 96 such layers, every word ends up enriched with the right context.
❓ Why AI needs this

Attention is the engine of modern AI:

  • Bahdanau attention (2014) — added to seq2seq, doubled translation quality overnight
  • Transformer (2017) — replaced RNNs with pure attention. "Attention Is All You Need."
  • BERT, GPT, T5, Llama — every modern LLM is a stack of attention + FFN blocks
  • Vision Transformers (ViT) — replace CNN feature extraction with attention on image patches
  • DALL-E, Stable Diffusion — cross-attention conditions image generation on text
  • Cost: O(N²·d) — N² because every token attends to every other
  • Linear/sparse attention research aims to scale beyond context length 100K

AI Deep Dive — Transformers

Topic 20
📚 The Concept

The transformer block has four key components. (1) Multi-head self-attention mixes information across positions — every token can look at every other (within the context window). (2) Position-wise FFN applies a 2-layer MLP independently to each token; this provides the per-token nonlinear processing that attention lacks. (3) Residual connections wrap each sublayer (attention and FFN) — without them, gradients vanish through deep stacks. (4) LayerNorm normalises each token's features (modern transformers prefer pre-norm: norm BEFORE the sublayer rather than after). Positional information is added to embeddings either as fixed sinusoidal positions (original) or learned relative biases (RoPE in modern LLMs). For autoregressive models (GPT-style), a causal mask in attention prevents tokens from peeking at future positions during training. The transformer's power: it's almost embarrassingly simple, fully parallel during training (unlike RNNs), and scales beautifully — losses follow predictable power laws as parameters and data grow ("scaling laws", Kaplan et al., 2020).

💡 Intuition: Each transformer layer does two things: attention shuffles information ACROSS tokens; FFN refines information WITHIN each token. Stack many such layers and you get a giant gossip network — each token gradually accumulates whatever context it needs from the rest of the sequence. By layer 96, "the" knows whether it's "the cat" or "the bank" or "the algorithm".
❓ Why AI needs this

Transformers are the dominant architecture across modalities since 2018:

  • Language: BERT, GPT-2/3/4, T5, Llama, Claude, Gemini — all transformer stacks
  • Vision: ViT, Swin, DETR — replaced or surpassed CNNs in classification, detection, segmentation
  • Speech: Whisper (OpenAI's STT) is a transformer encoder-decoder
  • Multimodal: CLIP, Flamingo, GPT-4V, Gemini — text + images via cross-attention
  • Code: Codex, AlphaCode, Code Llama — transformers fluent in Python, Rust, etc.
  • RL agents: Gato, Decision Transformer — even sequential decision-making is now transformers
  • Cost: O(N²·d) per layer — quadratic context length; flash-attention and sparse variants ease this

★ The Modern Recipe: AdamW optimiser + cosine schedule with linear warmup + LayerNorm (transformers) or BatchNorm (CNNs) + dropout (0.1–0.3) + L2/weight decay (0.01) + He init for ReLU + residual connections.
