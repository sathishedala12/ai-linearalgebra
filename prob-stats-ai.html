AI Deep Dive — Sample Spaces & Events

Topic 01
📚 The Concept

Three Kolmogorov axioms define probability. (1) P(A) ≥ 0 for any event A. (2) P(Ω) = 1 — the entire sample space has probability 1. (3) For disjoint events, P(A ∪ B) = P(A) + P(B). Everything else — conditional probability, Bayes, expectation, distributions — is downstream of these three. The sample space Ω is the set of all possible outcomes; an event is a subset of Ω; probability is a function from events to [0, 1] that respects the axioms.

💡 Intuition: A sample space is the "menu of everything possible." An event is checking a few items on that menu. Probability is the area on the menu those items occupy, when the whole menu has area 1.
❓ Why AI needs this

Every AI prediction is implicitly an event in some sample space. Defining that space precisely is half the modelling problem.

  • Classifier output: sample space = the set of class labels {cat, dog, …}
  • LLM next token: sample space = vocabulary (50K+ tokens)
  • Dropout: each neuron's "keep/drop" is one Bernoulli sample space
  • Reinforcement learning: states, actions, rewards each have their own sample space
⚙️ How AI uses it

Categorical sampling lives in torch.distributions:

probs = torch.softmax(logits, dim=-1) # Sample one outcome from the sample space: dist = torch.distributions.Categorical(probs) token = dist.sample() # Or measure probability of an event: p_top5 = probs.topk(5).values.sum()

Two fair dice. Sample space has 36 ordered outcomes (a, b) with a, b ∈ {1, …, 6}. Compute P(sum = 7) two ways: counting and a quick simulation.

AI Deep Dive — Random Variables

Topic 02
📚 The Concept

A random variable X is a function from the sample space Ω to ℝ. Discrete RVs take values in a countable set (integers, labels) and are described by a probability mass function P(X = x). Continuous RVs take values in a continuum (real numbers) and are described by a probability density function p(x), where probabilities are areas: P(a ≤ X ≤ b) = ∫ₐᵇ p(x) dx. Two RVs are identically distributed if they have the same PMF/PDF; they are independent if knowing one tells you nothing about the other.

💡 Intuition: "Random variable" sounds like a thing, but it's really a recipe — a function attached to a chance experiment that says "if outcome ω happens, the number is X(ω)." Whenever you write E[X] or Var(X), you're talking about that recipe.
❓ Why AI needs this

Every quantity inside a model is a random variable — when training, when sampling, when measuring uncertainty.

  • Weights initialised from N(0, σ²) → each weight is a continuous RV
  • Predicted class is a discrete RV taking values in the label set
  • Sampling temperature T scales logits before turning them into RVs
  • Diffusion models add Gaussian RV noise at every timestep
⚙️ How AI uses it

Both discrete and continuous RVs are first-class:

# Discrete RV: predicted class dist = Categorical(logits=model(x)) y_hat = dist.sample() # Continuous RV: weight init / latent code z = torch.randn(batch, latent_dim) # N(0, 1) # Reparameterized RV in VAE: z = mu + sigma * torch.randn_like(mu)

🧮 Worked Example

Let X = number of heads in 3 fair coin flips. List the PMF, compute E[X] and Var(X) directly from the definition.

PMF, mean, and variance of X = #heads in 3 flips
Given3 fair coin flips, X = number of heads, |Ω| = 8
Step 1P(X=0) = 1/8, P(X=1) = 3/8, P(X=2) = 3/8, P(X=3) = 1/8 (sums to 1 ✓)
Step 2E[X] = Σ x·P(X=x) = 0·1/8 + 1·3/8 + 2·3/8 + 3·1/8 = 12/8 = 1.5
Step 3E[X²] = 0 + 1·3/8 + 4·3/8 + 9·1/8 = 24/8 = 3
Step 4Var(X) = E[X²] − E[X]² = 3 − 2.25 = 0.75
ResultX ~ Binomial(3, 0.5) → mean = np = 1.5, variance = np(1−p) = 0.75. Direct PMF and the binomial formula agree.

Three ways to fully describe one random variable
📊

Plain English: The PMF (discrete) or PDF (continuous) is the "shape" — how likely each value is. The CDF F(x) = P(X ≤ x) tells you how much probability has accumulated by x. CDFs always go from 0 to 1, left to right. Sampling from a distribution is just inverse-CDF: draw u ∈ [0, 1] uniformly, then x = F⁻¹(u).

AI Deep Dive — PMF, PDF & CDF

Topic 03
📚 The Concept

PMF, PDF, and CDF are three lenses on the same random variable. The PMF works for discrete RVs and gives an actual probability at each value. The PDF works for continuous RVs and is a density — p(x) by itself isn't a probability (can be >1!) but its integral over an interval is. The CDF F(x) = P(X ≤ x) works for both, and it's the universal currency: the PDF is just F's derivative, and F⁻¹ is what every random sampler is implicitly computing. CDFs are also how we compare distributions — Kolmogorov-Smirnov tests look at max |F₁(x) − F₂(x)|.

💡 Intuition: PDF is "instantaneous likelihood per unit x" (think speed). CDF is accumulated probability up to x (think distance). They're related by the same calculus that links rates to totals.
❓ Why AI needs this

Every probabilistic model is fundamentally a PDF or PMF. Training maximises log-PDF on the data; sampling inverts the CDF.

  • Negative log-likelihood loss = −log p(x; θ) summed over data → minimised in training
  • Inverse-CDF sampling: u ~ Uniform → F⁻¹(u) gives a sample from the target
  • Quantile regression predicts F⁻¹(τ) instead of the mean — robust to outliers
  • Calibration: a well-calibrated classifier has F(predicted) ≈ uniform on test labels
⚙️ How AI uses it

PyTorch distributions exposes log-PDF and CDF directly:

d = torch.distributions.Normal(mu, sigma) log_p = d.log_prob(x) # log PDF F_x = d.cdf(x) # CDF sample = d.sample() # inverse-CDF under the hood nll = -log_p.mean() # NLL loss

Expectation & Variance

The center of a distribution and how widely it spreads
⚖️

Plain English: Expectation E[X] is the long-run average if you sampled X infinitely many times — the balance point of the PDF. Variance Var(X) measures how far values typically stray from that balance point. Squared distance, on average. Standard deviation σ = √Var puts that spread back into the original units. Training loss is an expectation; gradient descent is trying to minimise it.

AI Deep Dive — Expectation & Variance

Topic 04
📚 The Concept

Expectation is a linear operator: E[aX + bY] = aE[X] + bE[Y] for any RVs X, Y and constants a, b — even if X and Y are dependent. That linearity is doing enormous work in ML: it's why averaging mini-batch gradients is unbiased, why Monte Carlo estimators are unbiased, why E[loss] over a dataset can be split into per-example terms. Variance, by contrast, is non-linear: Var(X + Y) = Var(X) + Var(Y) only when X, Y are independent. The standard deviation σ has the same units as X — that's what makes it interpretable. The Cauchy-Schwarz inequality and Chebyshev's inequality are both expectation-based bounds that drop out of these definitions.

💡 Intuition: Expectation tells you where the center of mass sits; variance tells you how much the mass smears out. The bias-variance decomposition of generalisation error is exactly this split applied to a learning algorithm's predictions.
❓ Why AI needs this

Loss functions are expectations; optimisers minimise them. Gradient noise is a variance; reducing it speeds convergence.

  • Empirical risk minimisation: minimise (1/n)Σ ℓ(xᵢ; θ) ≈ E_x[ℓ(X; θ)]
  • Bias-variance tradeoff: test error = bias² + variance + irreducible noise
  • Adam's v_t tracks the second moment E[g²] of gradients
  • Reparameterization trick reduces variance of VAE/policy-gradient estimators
⚙️ How AI uses it

E[X] and Var(X) appear directly in optimiser state and statistics:

# Mini-batch loss ≈ E_x[ℓ(X; θ)] loss = ℓ(model(x), y).mean() loss.backward() # Adam tracks first moment (mean) and second moment # m_t ≈ E[g_t], v_t ≈ E[g_t²] # Variance reduction (control variates): # E[f(X) - g(X)] + E[g(X)] with chosen g

🧮 Worked Example

For Bernoulli(p): compute E[X] and Var(X). When is variance maximal?

Bernoulli mean and variance
GivenX ∈ {0, 1}, P(X=1) = p, P(X=0) = 1 − p
Step 1E[X] = 0·(1−p) + 1·p = p
Step 2E[X²] = 0²·(1−p) + 1²·p = p (since X² = X for Bernoulli)
Step 3Var(X) = E[X²] − E[X]² = p − p² = p(1 − p)
Maxd/dp [p(1−p)] = 1 − 2p = 0 → p = 0.5, max variance = 0.25
ResultA fair coin (p = 0.5) is the most uncertain Bernoulli. This is also why dropout uses p ≈ 0.5: maximum injected noise per neuron.

AI Deep Dive — Conditional & Independence

Topic 05
📚 The Concept

P(A | B) = P(A ∩ B) / P(B), defined whenever P(B) > 0. It "zooms in" on B and renormalises so the new sample space has total probability 1. Independence is the special case where conditioning doesn't change anything: A ⊥ B iff P(A | B) = P(A) iff P(A ∩ B) = P(A)P(B). The chain rule P(X₁, …, Xₙ) = P(X₁) · P(X₂ | X₁) · … · P(Xₙ | X₁, …, X_{n−1}) decomposes any joint distribution into a sequence of conditionals — which is exactly how an autoregressive language model factors the probability of a sentence. Conditional independence X ⊥ Y | Z is the workhorse of graphical models: it says Z "shields" X from Y.

💡 Intuition: "Given B" is a verbal restriction of the universe to where B is true. Independence says: the universe-restriction didn't matter. Most useful AI structure (HMMs, Bayes nets, transformers) is built from carefully chosen conditional-independence assumptions.
❓ Why AI needs this

Conditioning is how every model handles "given the input, predict the output." It's the entire point of supervised learning.

  • Classifiers learn P(y | x) — the conditional distribution of label given features
  • LLMs learn P(token_t | tokens_{<t}) — chain rule applied to text
  • Naive Bayes assumes features are conditionally independent given the label
  • Causal inference: P(Y | do(X)) is conditioning on an intervention, not observation
⚙️ How AI uses it

The chain rule is the structural identity that makes autoregressive LLMs work:

🧮 Worked Example

The base-rate fallacy. Disease prevalence 0.1%, test sensitivity 99%, specificity 95%. A patient tests positive — what's the actual probability they're sick?

P(disease | positive test)
GivenP(D) = 0.001, P(+ | D) = 0.99, P(+ | ¬D) = 0.05
Step 1P(+ ∩ D) = P(+ | D) · P(D) = 0.99 · 0.001 = 0.00099
Step 2P(+ ∩ ¬D) = P(+ | ¬D) · P(¬D) = 0.05 · 0.999 = 0.04995
Step 3P(+) = P(+ ∩ D) + P(+ ∩ ¬D) = 0.00099 + 0.04995 = 0.05094
Step 4P(D | +) = P(+ ∩ D) / P(+) = 0.00099 / 0.05094 ≈ 0.0194
ResultOnly ~1.94% chance you actually have the disease! False positives (5% of 99.9% of people) swamp the true positives (99% of 0.1%). The base rate dominates.

AI Deep Dive — Bernoulli & Binomial

Topic 06
📚 The Concept

Bernoulli(p) is the simplest non-trivial distribution: a single 0/1 outcome with probability p of being 1. Binomial(n, p) is what you get when you sum n independent Bernoulli(p) variables — the count of "successes" in n trials. The Binomial PMF P(X = k) = C(n, k) pᵏ (1−p)ⁿ⁻ᵏ has three pieces: the combinatorial coefficient C(n, k) counting which trials were successes, pᵏ for those k successes, and (1−p)ⁿ⁻ᵏ for the n−k failures. Mean np and variance np(1−p) follow from linearity of expectation applied to the Bernoulli sum. As n grows with p fixed, Binomial converges to Normal (CLT); as n grows with np fixed, it converges to Poisson (rare-event regime).

💡 Intuition: Bernoulli = "yes/no question." Binomial = "how many yes-es out of n questions?" These two cover an enormous fraction of real-world counting situations, from A/B test conversions to spam-filter precision.
❓ Why AI needs this

Binary classification, dropout, and many evaluation metrics are Bernoulli/Binomial under the hood.

  • Sigmoid output of a binary classifier IS the parameter p of a Bernoulli
  • BCEWithLogitsLoss = negative log-likelihood under Bernoulli
  • Dropout: each neuron Bernoulli(p_keep); a layer's kept-count is Binomial
  • Accuracy on a test set: #correct out of n is Binomial — bounds the eval uncertainty
⚙️ How AI uses it

Binary cross-entropy IS Bernoulli negative log-likelihood:

p = torch.sigmoid(logit) # Bernoulli parameter # NLL of label y under Bernoulli(p): loss = -(y*torch.log(p) + (1-y)*torch.log(1-p)) # Equivalently: loss = F.binary_cross_entropy_with_logits(logit, y) # Dropout: Bernoulli mask mask = torch.bernoulli(torch.full_like(x, 0.8))

AI Deep Dive — Categorical & Softmax

Topic 07
📚 The Concept

A Categorical(p₁, …, p_K) distribution is fully specified by K non-negative numbers that sum to 1. Softmax is the canonical way to parameterise such a distribution from K free real numbers (logits). Three properties make softmax the right choice. (1) Translation invariance: softmax(z + c) = softmax(z). (2) Order-preserving: rankings of zᵢ and pᵢ match. (3) Smooth and differentiable everywhere — argmax isn't, which is why argmax can't be backpropagated through but softmax can. The temperature T is a knob that interpolates between argmax (T → 0) and uniform (T → ∞), giving exact control over how peaky the distribution is — top-p, top-k, and nucleus sampling all play in this temperature regime.

💡 Intuition: Logits are "scores." Softmax converts scores into a probability budget — the difference between two scores controls how much budget separates them, exponentially. Doubling the gap squares the relative odds.
❓ Why AI needs this

Multi-class classification, language modelling, attention, and RL policies all output Categorical distributions parameterised by softmax.

  • Output layer for K-class classifier: linear → softmax → cross-entropy loss
  • Each LLM step samples from softmax(logits / T) over the vocabulary
  • Attention weights are softmax over similarity scores
  • Policy gradient RL: π(a | s) = softmax of action-value scores
⚙️ How AI uses it

Always use the numerically stable forms (log-softmax, F.cross_entropy):

probs = F.softmax(logits, dim=-1) log_probs = F.log_softmax(logits, dim=-1) # Cross-entropy combines both stably: loss = F.cross_entropy(logits, labels) # Sample with temperature: dist = Categorical(logits=logits / T) token = dist.sample()

AI Deep Dive — Gaussian

Topic 08
📚 The Concept

The Normal distribution N(μ, σ²) has PDF p(x) = (2πσ²)⁻¹/² exp(−(x−μ)² / (2σ²)). Three remarkable properties make it ubiquitous. (1) Closure under linear operations: if X ~ N(μ, σ²), then aX + b ~ N(aμ + b, a²σ²). Sums of independent Normals are Normal. (2) Maximum entropy among distributions with fixed mean and variance — it's the "most uncertain" choice given those constraints. (3) Central Limit Theorem: sums of many independent variables — regardless of their original shape — approach a Normal. The multivariate generalisation N(μ, Σ) keeps all these properties and parameterises a covariance matrix Σ that controls correlations between dimensions.

💡 Intuition: The Normal is the lazy, default, "no further information" choice for a continuous quantity. That's not a bug; the maximum-entropy property says it's mathematically the most honest distribution consistent with knowing only the mean and variance.
❓ Why AI needs this

  • VAE latent z ~ N(0, I); reparameterisation: z = μ + σ⊙ε with ε ~ N(0, I)
  • Diffusion: each forward step adds N(0, β_t I) noise to the data
  • MSE loss = NLL under N(prediction, σ²) — least-squares is Gaussian MLE

AI Deep Dive — Joint Distributions

Topic 11
📚 The Concept

Two RVs X, Y have a joint p(x, y) over the product space. From it you recover the marginal by summing out: p(x) = Σ_y p(x, y) (or integrating). The conditional is p(x | y) = p(x, y) / p(y) — and rearranging gives the chain rule p(x, y) = p(y) · p(x | y) = p(x) · p(y | x), which is what makes Bayes' theorem work. The bivariate Normal shows the geometry beautifully: contour ellipses tilted by correlation ρ. Conditioning on Y = y slices the ellipse and gives another Normal, with mean shifted along the regression line and variance shrunk by (1 − ρ²). This shrinkage is "explained variance" — R² in linear regression literally equals ρ². For more than two variables, the joint factorises along whatever conditional-independence structure your model assumes (Naive Bayes, Bayes nets, HMMs, transformers).

💡 Intuition: A joint distribution is a 2D heatmap. Marginalising = projecting onto one axis. Conditioning = slicing along a fixed value of the other axis and re-normalising. The slice is always thinner than the projection — that thinning is the information gain.
❓ Why AI needs this

Every supervised learner targets either p(y | x) (discriminative) or p(x, y) (generative). The factorisation choice is the modelling choice.

  • Discriminative models (logistic regression, neural nets) parameterise p(y | x) directly
  • Generative models (Naive Bayes, GMM, diffusion) parameterise p(x, y) or p(x)
  • Latent variable models: p(x) = ∫ p(x | z) p(z) dz — marginalise out z
  • Multivariate Gaussians: covariance matrix = full joint structure for Normal data
⚙️ How AI uses it

AI Deep Dive — Bayes' Theorem

Topic 12
📚 The Concept

Bayes' theorem rewrites the joint p(H, E) two ways and equates them: p(H, E) = p(H) p(E | H) = p(E) p(H | E). Solve for p(H | E):   p(H | E) = p(E | H) p(H) / p(E). The pieces have names. Prior p(H) — what you believed about H before seeing E. Likelihood p(E | H) — how plausible the evidence is under each hypothesis. Marginal evidence p(E) = ∫ p(E | H) p(H) dH — a normalising constant. Posterior p(H | E) — your updated belief. The deep insight: posterior ∝ likelihood × prior, with the evidence acting only as a normaliser. Multiplying repeatedly is sequential Bayesian updating — every new observation moves the posterior closer to the truth, weighted by how informative each observation is. With Gaussian likelihoods and priors, posteriors stay Gaussian and precisions (1/σ²) add cleanly.

💡 Intuition: Prior is "what I knew." Likelihood is "what this evidence says, assuming each hypothesis." Posterior is the multiplied compromise. The stronger the prior or the more decisive the likelihood, the more it dominates the product.
❓ Why AI needs this

Bayesian methods sit underneath probabilistic ML, regularisation theory, and the entire workflow of model uncertainty.

  • L2 regularisation = MAP under a Gaussian prior on weights
  • L1 regularisation = MAP under a Laplace prior (sparsity-inducing)
  • Bayesian neural nets place priors on weights, infer posteriors from data
  • Naive Bayes classifier: P(class | features) ∝ P(features | class) P(class)
  • Spam filtering, fraud detection, medical AI all rest on Bayes

AI Deep Dive — Entropy & Information

Topic 13
📚 The Concept

Entropy H(X) = −E[log p(X)] = −Σ p(x) log p(x) measures the average information content of one sample from p. The information of a single outcome with probability p is −log p — rare outcomes carry more bits. Three milestone properties. (1) H(X) ≥ 0 with equality iff X is deterministic. (2) Among distributions on K outcomes, the uniform distribution maximises H at log K. (3) Conditioning never increases entropy: H(X | Y) ≤ H(X), with equality iff X ⊥ Y. Mutual information I(X; Y) = H(X) − H(X | Y) measures how much knowing Y reduces uncertainty in X — a non-negative, symmetric, unit-free measure of dependence. Differential entropy generalises this to continuous RVs, with one quirk: differential entropy can be negative.

💡 Intuition: Entropy answers "how many yes/no questions, on average, to learn the outcome?" A fair die has log₂ 6 ≈ 2.58 bits — you can pin it down in roughly 3 well-chosen questions. Mutual information says "how many questions does Y save you when guessing X?"

  • Information bottleneck: compress X but preserve I(Z; Y) for the task

AI Deep Dive — KL & Cross-Entropy

Topic 14
📚 The Concept

KL divergence D_KL(P ‖ Q) = E_P[log(P/Q)] measures the expected number of extra bits needed to encode samples from P using a code optimised for Q. It's not symmetric. "Forward KL" D_KL(P ‖ Q) penalises Q being zero where P has mass — Q is forced to be mode-covering, spreading itself to cover all of P's modes. "Reverse KL" D_KL(Q ‖ P) penalises Q having mass where P is zero — Q becomes mode-seeking, latching onto a single mode. This asymmetry is the core distinction between maximum-likelihood training (forward KL) and variational inference (reverse KL). Cross-entropy H(P, Q) = H(P) + D_KL(P ‖ Q): since H(P) doesn't depend on Q, minimising cross-entropy is equivalent to minimising KL — that's why all classifiers optimise cross-entropy. Jensen-Shannon divergence symmetrises KL; the f-divergence family generalises both.

💡 Intuition: Cross-entropy is a "bad encoder" cost: how many bits per sample do you waste using Q when P is the truth? KL is the wasted-bits-only part. Two distributions agree fully → no waste, KL = 0.
❓ Why AI needs this

KL and cross-entropy are the loss functions of modern ML.

  • VAE objective: ELBO = E[log p(x | z)] − KL(q(z | x) ‖ p(z)) — both directions appear
  • RLHF / DPO / PPO: KL penalty keeps fine-tuned policy near reference policy
  • Knowledge distillation: student matches teacher's softmax — minimise KL(teacher ‖ student)
  • GAN's f-divergence variants generalise vanilla JS-divergence GANs

AI Deep Dive — Maximum Likelihood

Topic 15
📚 The Concept

MLE is a frequentist point estimator: pick the θ that maximises the likelihood L(θ) = ∏ p(xᵢ | θ). Working in log-space turns product into sum and avoids underflow: log L(θ) = Σ log p(xᵢ | θ). MLE has lovely asymptotic guarantees: under mild conditions it's consistent (θ̂ → θ_true as n → ∞), asymptotically efficient (lowest possible variance, attaining the Cramér-Rao bound), and asymptotically Normal (θ̂ ≈ N(θ_true, I⁻¹/n) where I is the Fisher information). On finite samples, MLE can be biased — for example, the MLE of σ² in a Gaussian uses 1/n, but the unbiased estimator uses 1/(n−1) (Bessel's correction). MLE also overfits when models are flexible: maximising the likelihood without regularisation is the cause of overfitting, fixed in topic 16 by adding a prior.

💡 Intuition: MLE asks: "given my data, which parameter makes this exact data set most plausible?" When the answer is the sample mean, sample variance, or the proportion that worked, that's MLE giving you the obvious estimator — which is exactly why those classical estimators are so common.
❓ Why AI needs this

Most loss functions in ML are negative log-likelihoods under specific distributional assumptions.

  • MSE = NLL under Gaussian noise: MLE of regression weights is least-squares
  • Cross-entropy = NLL under Categorical labels: MLE of classifier params
  • Logistic regression = MLE of Bernoulli probabilities given features
  • Language modelling = MLE on next-token Categorical, summed over corpus
⚙️ How AI uses it

AI Deep Dive — MAP & Priors

Topic 16
📚 The Concept

MAP estimation maximises the posterior p(θ | data) ∝ p(data | θ) p(θ). Taking logs: argmax of [log p(data | θ) + log p(θ)]. The first term is the log-likelihood (drives MLE); the second is the log-prior (the regulariser). The crucial AI insight is the exact correspondence between regularisation and Bayesian priors. Adding λ‖w‖² / 2 to the loss is mathematically identical to assuming a Gaussian prior w ~ N(0, 1/λ). Adding λ‖w‖₁ corresponds to a Laplace prior, which has a sharp peak at zero and is responsible for the sparsity-inducing behaviour of L1. Heavy-tailed priors (e.g. Student-t, Cauchy) yield robust estimators that ignore outliers. As n → ∞, the prior contribution becomes negligible compared to the likelihood — MAP → MLE — which is why with enough data, the regulariser barely matters.

💡 Intuition: A prior is "data you brought from home." It pulls your estimate toward your prior beliefs in proportion to its strength. The strength is measured in pseudo-counts — Beta(2, 2) is "I've already seen one of each" — so 1 real flip plus this prior is effectively 3 observations.
❓ Why AI needs this

Every weight regulariser is a MAP prior. Choosing one IS doing Bayesian modelling.

  • Weight decay (AdamW): Gaussian prior on w → L2 regulariser → MAP
  • L1 / Lasso: Laplace prior → induces sparsity → feature selection
  • Dropout as Bayesian approximation (Gal & Ghahramani, 2016)
  • Label smoothing: Dirichlet prior on label distribution → softer targets
  • "Pretrain + fine-tune" is informally MAP: pretrained weights are a prior
⚙️ How AI uses it

AI Deep Dive — Bayesian Inference

Topic 17
📚 The Concept

Full Bayesian inference produces a distribution over parameters, not a point estimate. Posterior: p(θ | D) ∝ p(D | θ) p(θ). For most realistic models the marginal evidence p(D) = ∫ p(D | θ) p(θ) dθ is intractable, so we use one of three strategies: (1) conjugate priors — closed-form updates (Beta-Binomial, Normal-Normal, Dirichlet-Multinomial). (2) MCMC — Metropolis-Hastings, Hamiltonian Monte Carlo, NUTS — sample from the posterior without ever computing p(D). (3) Variational inference (VI) — pick a tractable family q(θ; φ) and minimise KL(q ‖ posterior) — turns inference into optimisation; this is what VAEs do. The headline benefit: predictions made by integrating over the posterior — the posterior predictive p(x_new | D) = ∫ p(x_new | θ) p(θ | D) dθ — incorporate parameter uncertainty automatically. They're calibrated; they don't pretend they know more than they do.

💡 Intuition: MLE/MAP say "the answer is θ̂." Bayesian says "the answer is this whole cloud of plausible θ, weighted by how well each fits the data." For prediction, the cloud generates a wider, calibrated forecast — not just the single best guess.
❓ Why AI needs this

Calibrated uncertainty matters wherever decisions cost real money — medical, autonomous driving, active learning, RL exploration.

  • Bayesian neural nets give per-prediction uncertainty (Gal & Ghahramani 2016 with dropout)
  • Active learning: query points where posterior predictive uncertainty is highest
  • Bandit / Thompson sampling: action drawn from posterior over reward params
  • Bayesian optimisation (BO) for hyperparameter search uses GP posteriors
  • Modern LLM uncertainty quantification uses MC dropout or deep ensembles as Bayesian approximations

AI Deep Dive — Sampling Methods

Topic 18
📚 The Concept

Monte Carlo replaces expectations with sample averages: E[f(X)] ≈ (1/n) Σ f(xᵢ). The estimator is unbiased and the law of large numbers gives convergence; the central limit theorem gives error bars of order 1/√n — a remarkable feature, because this rate doesn't depend on dimension. Different problems need different sampling techniques. Inverse-CDF sampling works when F⁻¹ is easy. Rejection sampling wraps a tight envelope M·g(x) around the target; acceptance rate is 1/M. MCMC (Metropolis-Hastings, Gibbs, HMC, NUTS) builds a Markov chain whose stationary distribution is the target — needs only an unnormalised target density, perfect for posteriors. Quasi-Monte Carlo uses low-discrepancy sequences instead of random samples for better deterministic error rates.

💡 Intuition: Sampling turns a hard integral into many easy operations. The price you pay is variance, but the law of large numbers gives 1/√n convergence in any dimension — better than grid integration as soon as d ≥ 4.
❓ Why AI needs this

Sampling is everywhere: VAEs, diffusion, RL policies, LLM decoding, Bayesian inference, even gradient estimators.

  • VAE/policy gradient: reparameterised samples for low-variance gradients
  • Diffusion models: every generated image is the end of a long sampling chain
  • LLM decoding: top-k, top-p, nucleus sampling are all sample-from-softmax variants
  • RL exploration: Thompson sampling draws θ from the posterior to pick actions
  • Bayesian neural nets / probabilistic programs: posterior MCMC via NumPyro, Stan

AI Deep Dive — LLN & CLT

Topic 19
📚 The Concept

Both theorems concern X̄_n = (1/n) Σ Xᵢ for i.i.d. samples Xᵢ with mean μ and finite variance σ². Law of Large Numbers says X̄_n → μ — the sample mean converges to the true mean. There are two flavours: weak LLN (convergence in probability) and strong LLN (almost-sure convergence). LLN justifies estimating expectations by averages, which underpins all of empirical risk minimisation. Central Limit Theorem goes further: it characterises how fast X̄_n converges. The standardised sample mean √n (X̄_n − μ) / σ converges in distribution to N(0, 1), regardless of the source distribution's shape. Two consequences: SE = σ/√n shrinks at the universal "square-root rate," and approximate confidence intervals X̄_n ± 1.96·SE work for almost any parent distribution given enough data. CLT extends to martingales (martingale CLT) and to dependent data (mixing CLT) — both used in modern theory of stochastic gradient descent.

💡 Intuition: LLN says "averages stabilise." CLT says "averages stabilise in a Gaussian-shaped way." Together they explain why almost every well-behaved measurement in science looks Gaussian, and why the standard error always looks like σ/√n.
❓ Why AI needs this

Mini-batch SGD, evaluation metrics, model averaging, and bootstrap CIs all rely on LLN/CLT.

  • Bootstrap confidence intervals: resample to get SE without parametric assumptions
  • Deep ensemble averaging stabilises predictions via LLN
  • SGD convergence theory uses martingale CLT under stochastic gradient noise

AI Deep Dive — Hypothesis Testing

Topic 20
📚 The Concept

Hypothesis testing is decision-making under uncertainty. Set up the null hypothesis H₀ (the conservative default — "the new model is no better than the old one") and alternative H₁. Compute a test statistic T from data; under H₀ it has a known distribution. The p-value is P(|T| ≥ |t_observed| | H₀) — the probability of seeing data this extreme or more if H₀ were true. Reject H₀ at level α when p < α; α is the false-positive rate (Type I error) you're willing to tolerate. The power 1 − β is the probability of correctly rejecting H₀ when H₁ is true; it grows with sample size and effect size, and shrinks with stricter α. Modern alternatives include confidence intervals (often more informative than p-values), Bayesian posteriors (give P(H₁ | data) directly), and the false-discovery rate (FDR / Benjamini-Hochberg) for many simultaneous tests.

💡 Intuition: A p-value is "if there were really no effect, how surprising would these data be?" Small p = surprising under H₀ = evidence against H₀. It's NOT P(H₀ | data), and it's NOT effect size — only a measure of how compatible the data are with the null world.
❓ Why AI needs this

Every model comparison, A/B test, and "is this regression coefficient real?" question is hypothesis testing.

  • Comparing model accuracies: McNemar's test for paired predictions
  • Multiple comparisons (e.g. ablation studies): Bonferroni or FDR correction
  • Permutation tests: resample labels to build the null distribution
  • Sequential testing for early-stop in long experiments (mSPRT, always-valid p-values)
